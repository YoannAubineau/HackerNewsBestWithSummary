## Résumé de l'article

Anthropic a confirmé auprès de son personnel que l'utilisation de Claude CLI via OpenClaw est à nouveau autorisée. OpenClaw supporte maintenant à la fois l'accès par clé API Anthropic et la réutilisation de Claude CLI, avec plusieurs configurations et optimisations disponibles.

- OpenClaw traite l'utilisation de Claude CLI et la commande `claude -p` comme sanctionnées à moins qu'Anthropic ne publie une nouvelle politique
- Plusieurs méthodes d'authentification sont supportées : clé API Anthropic, CLI Anthropic, et jetons OAuth legacy
- Les modèles Claude supportent le cache de prompts (5 minutes par défaut avec clé API), la fenêtre de contexte 1M en bêta, et les niveaux de réflexion adaptatifs
- Le mode "fast" d'OpenClaw permet de router le trafic direct vers api.anthropic.com avec gestion automatique des niveaux de service
- Pour les déploiements en production, les clés API Anthropic restent le chemin recommandé pour un contrôle explicite de la facturation et une disponibilité continue

## Discussion sur Hacker News

**Confirmations** :
- La stratégie de communication d'Anthropic est confuse et peu professionnelle : annonces via Twitter ou postes Reddit au lieu de canaux officiels, créant de l'incertitude persistante chez les utilisateurs.
- Les restrictions répétées sur Claude CLI et les changements de politique imprévisibles ont endommagé la confiance des utilisateurs fidèles, les poussant à tester des alternatives (Codex, Kimi, GLM) malgré la qualité perçue de Claude.
- Les utilisateurs ressentent une hausse des prix combinée à une réduction des limites d'usage, interprétée comme une « enshittification » progressive du service malgré la goodwill initial d'Anthropic.

**Réfutations** :
- La distinction entre utiliser OpenClaw au sein d'une session Claude Code officielle (autorisé) versus détourner les credentials OAuth (facturation API supplémentaire) est légitime et conforme aux ToS, selon les défenseurs d'Anthropic.
- L'argument que les fournisseurs n'ont aucun « moat » est contesté : les données montrent une préférence utilisateur réelle pour différents modèles selon les tâches (Opus pour frontend, GPT 5.3+ pour backend), suggérant une différenciation durable plutôt qu'une interchangeabilité complète.
- Les hausses de prix et restrictions reflètent une contrainte réelle de compute, nécessaire pour atteindre la rentabilité d'un service qui brûle massivement du capital, et ne constituent pas une malveillance délibérée.
