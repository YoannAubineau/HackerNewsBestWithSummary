## Discussion sur Hacker News (66 commentaires analysés)

**Avis positifs** :
- Les préoccupations concernant la divulgation du code généré par l'IA sont justifiées : le secret autour de l'utilisation de Claude Code pose problème de transparence, notamment dans un contexte open source où la qualité du code doit être vérifiable.
- Le modèle open-core peut être légitime, mais les incitations perverses qu'il crée (suppression d'informations sur les alternatives, promotion de produits fermés) soulèvent des questions éthiques valides quand les mainteneurs du cœur détiennent aussi ces couches commerciales.
- MeshCore et Meshtastic offrent une véritable utilité pratique malgré leurs défauts : des milliers de nœuds fonctionnels, des cas d'usage réels (jeux collaboratifs, réseaux de capteurs) et des alternatives open source disponibles.

**Avis négatifs** :
- Juger le code uniquement sur ses "mérites" ignore la nature de l'IA : elle excelle à produire du code plausible mais incorrect, rendant l'évaluation difficile sans oracles de validation (tests automatisés, comparaison comportementale), que MeshCore reconnaît elle-même ne pas avoir.
- Les problèmes soulevés dépassent largement l'IA : qualité de code généralement faible (pas de tests, acceptation de coordonnées GPS invalides, paramètres radio illégaux ignorés), et une équipe core plus intéressée par des narratifs anti-IA que par l'amélioration technique.
- Le conflit de marque semble être un symptôme d'un problème structurel plus large : manque de gouvernance claire, absence de code source facilement accessible pour l'app officielle, et un projet apparemment fermé à la critique externe malgré son statut "open source".
