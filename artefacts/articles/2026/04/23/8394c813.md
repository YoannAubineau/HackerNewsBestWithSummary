## Discussion sur Hacker News (38 commentaires analysés)

**Avis positifs** :
- L'approche "AI-first" différencie Tolaria d'Obsidian en positionnant l'IA comme agent autonome avec accès au système de fichiers, plutôt que comme simple assistant UX, ouvrant des possibilités de design nouvelles.
- L'intégration native de Git comme couche de synchronisation et contrôle de version permet de tracer les modifications de l'IA comme contributeur à part entière, offrant transparence et auditabilité.
- L'interface est magnifique, la gestion de 10k notes sans dégradation performative est impressionnante, et des fonctionnalités essentielles (collage d'images, organisation par types et relations) fonctionnent bien d'emblée.
- Le choix de Tauri pour contourner les limites de Swift sur l'éditeur Markdown est pragmatique et offre de bonnes performances tout en restant open-source.
- Les solutions créatives pour le mobile (capture via Telegram avec OpenClaw) et la synchronisation multi-appareils via markdown + git montrent une philosophie cohérente.

**Avis négatifs** :
- L'absence de version native macOS (utilisation d'une web app Tauri) et surtout l'absence de version mobile déçoivent les utilisateurs habitués à Obsidian, tuant le potentiel comme outil quotidien malgré les qualités du desktop.
- Les bugs détectés dès la sortie (tri par date modifiée cassé après commit Git, raccourcis clavier manquants comme Ctrl+E) et limitations d'édition (``` ne crée pas automatiquement de bloc de code) suggèrent une version trop précoce pour usage sérieux.
- L'affichage de l'IA comme "first-class citizen" au détriment de l'utilisateur humain suscite de la réticence philosophique : pourquoi privilégier les agents à l'IA quand l'outil devrait servir les humains ?
- Le risque classique de "logiciel jetable d'une personne" persiste, malgré le contexte open-source, et le paradoxe d'une meilleure organisation préexistante (vs. capture) par rapport aux véritables besoins d'utilisateurs.
