## Résumé de l'article

L'API Prompt est une nouvelle interface web proposée par Chrome (Blink) pour permettre aux développeurs d'afficher des boîtes de dialogue de saisie utilisateur. Mozilla évalue actuellement cette proposition dans le cadre du processus de normalisation des standards web.

- Chrome (Blink) a publié une intention de prototype (intent-to-prototype) pour l'API Prompt
- Mozilla est en phase d'examen de la proposition sans position officielle publiée pour le moment
- La proposition manque actuellement de documentation explicitée (explainer, URLs de référence)
- La position de WebKit n'a pas été communiquée publiquement sur cette API

## Discussion sur Hacker News (125 commentaires)

**Avis positifs** :
- Chrome a historiquement amélioré l'expérience web en forçant les standards et les optimisations, notamment en sortant des cauchemars du navigateur IE
- Une API standardisée pour les modèles locaux pourrait améliorer la confidentialité en évitant l'envoi de données vers des serveurs externes, contrairement aux appels API traditionnels
- Le processus de développement a suivi une trajectoire normale (Dev Trial puis Origin Trial) avec collecte de retours et itérations sur le modèle et les fonctionnalités
- D'autres plateformes (iOS, macOS, Windows) intègrent déjà des APIs d'accès aux modèles IA, créant une attente industrielle légitime
- L'API encourage les développeurs à construire des améliorations progressives plutôt que des dépendances dures, avec des chemins d'erreur clairs

**Avis négatifs** :
- Google abuse sa position dominante pour imposer une API propriétaire de facto, forçant Mozilla et Apple à cloner un modèle Google ou perdre en compatibilité
- La politique d'utilisation prohibée imposée par Google sur une API web (restrictions sur le contenu politique, sexuel, etc.) viole les principes de liberté d'expression et pose un précédent dangereux
- Les développeurs risquent de se verrouiller sur les quirks spécifiques du modèle Gemini de Google, rendant les futures mises à jour du modèle difficiles pour tous les navigateurs
- Aucune demande réelle des utilisateurs n'existe : l'intégration IA est imposée par défaut sans consentement, téléchargeant silencieusement 4-7 GB de modèles et consommant des ressources massives sur du matériel grand public
- Les signaux de développeurs en faveur de cette API sont mixtes voire polarisés, et les preuves présentées par Chrome étaient insuffisantes au moment du lancement
