## Résumé de l'article

Zed permet désormais d'orchestrer plusieurs agents s'exécutant en parallèle dans la même fenêtre, avec une nouvelle Threads Sidebar pour contrôler l'accès aux dossiers et référentiels, et monitorer leur activité en temps réel à 120 fps.

- La Threads Sidebar regroupe les threads par projet et permet de combiner différents agents, de travailler entre plusieurs projets et d'isoler les worktrees selon les besoins
- L'interface a été repensée avec la Threads Sidebar ancrée à gauche par défaut pour placer les threads au centre du flux de travail
- La fonction repose sur le concept d'« agentic engineering » : combiner l'assistance IA avec l'engagement direct du développeur dans le code plutôt que de générer automatiquement des lignes de code
- Les opérations courantes comme arrêter, archiver ou lancer des threads sont accessibles instantanément depuis la Sidebar
- La fonctionnalité est disponible dans la dernière version de Zed, accessible via l'icône en bas à gauche ou les raccourcis clavier option-cmd-j (macOS) et ctrl-option-j (Linux/Windows)

## Discussion sur Hacker News (47 commentaires analysés)

**Avis positifs** :
- Zed propose une implémentation d'agents parallèles plus logique que ses concurrents (Warp, Conductor), avec support agent-agnostic, worktrees multiples, et une UI de qualité supérieure aux wrappers CLI.
- Les hooks de cycle de vie des worktrees (création/suppression) offrent des possibilités puissantes : configuration locale, duplication de bases de données, isolation automatique—une vraie avancée par rapport aux outils existants.
- Zed reste un excellent éditeur rapide et puissant en lui-même, avec des fonctionnalités de base solides et la possibilité de désactiver complètement les features IA si souhaité.
- Le système de threads parallèles améliore le workflow en permettant de basculer entre investigations secondaires sans perdre le contexte du travail principal, plutôt que simplement "paralléliser".

**Avis négatifs** :
- La nouvelle disposition par défaut repousse l'arborescence de fichiers et le code sur les côtés pour privilégier le panneau d'agents, ce qui déplaît à de nombreux utilisateurs et risque de repousser les nouveaux venus cherchant un éditeur classique.
- Zed manque de fonctionnalités importantes par rapport à Claude et ses concurrents : pas d'intégration MCP, absence de plugins/skills, et outils de review de code insuffisants pour valider des changements générés par agents.
- Les utilisateurs expriment des préoccupations quant au pivot stratégique vers les agents aux dépens de l'expérience d'édition traditionnelle : plusieurs QoL manquent (git avancé, débogage TS lent, configuration fine de l'UI).
- Les agents parallèles créent une dette cognitive importante et ne conviennent pas à tous les workflows ; beaucoup préfèrent revoir manuellement chaque modification pour éviter que les agents n'inscrivent des bugs dans le code ou ne compliquent inutilement l'architecture.
