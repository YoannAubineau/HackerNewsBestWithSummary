## Résumé de l'article

Anthropic a enquêté sur les rapports de dégradation de qualité de Claude et identifié trois problèmes distincts affectant Claude Code, l'SDK Claude Agent et Claude Cowork, tous résolus au 20 avril. L'API n'a pas été impactée.

- **Effort par défaut inapproprié** : Le passage du réglage d'effort à "medium" pour Opus 4.6 a causé des latences excessives et une apparence gelée de l'interface ; Anthropic a rétabli le défaut à "xhigh" pour Opus 4.7 et "high" pour les autres modèles le 7 avril.

- **Bug de cache de raisonnement** : Une implémentation défectueuse de l'en-tête API `clear_thinking_20251015` effaçait l'historique de raisonnement à chaque tour au lieu d'une seule fois, causant l'oubli, la répétition et des choix d'outils aberrants ; corrigé le 10 avril en v2.1.101.

- **Limite de longueur excessive dans le prompt système** : Une instruction ajoutée pour limiter la sortie à ≤25 mots entre les appels d'outils a réduit l'intelligence de 3% ; revertée le 20 avril après détection lors d'une évaluation plus large.

- **Mesures futures** : Anthropic utilisera les builds publics internes, améliorera l'outil de révision de code, renforcera les contrôles sur les changements de prompt système, et effectuera des tests plus complets avant les déploiements. Les limites d'utilisation ont été réinitialisées pour tous les abonnés.

## Discussion sur Hacker News (62 commentaires analysés)

**Avis positifs** :
- Anthropic a reconnu les trois bugs majeurs, les a corrigés et offert des remboursements, montrant une certaine transparence et responsabilité face aux problèmes de qualité.
- Boris de l'équipe Claude Code s'engage activement dans les commentaires pour expliquer les décisions techniques (notamment le dilemme latence vs. qualité des sessions inactives) et promesse d'investir davantage dans la robustesse.
- Pour les utilisateurs toujours satisfaits, Claude reste supérieur à la concurrence en termes de qualité brute malgré ces incidents, avec des cas d'usage où Opus 4.7 surpasse GPT-5.4.

**Avis négatifs** :
- Accumulation de bugs graves (prompt système réduit, suppression de tokens de réflexion, inversions de paramétrages) suggère une culture de déploiement hâtif et un manque de contrôle qualité robuste, particulièrement préoccupant pour un produit premium payant.
- Les explications données semblent des prétextes pour des optimisations de coûts (purge des tokens à 1h correspondant à la limite de cache, modification du prompt pour réduire la verbosité) plutôt que des améliorations d'expérience utilisateur, alimentant la méfiance envers les communications d'Anthropic.
- Absence totale de communication proactive sur les changements de système prompt ou de variables de modèle, contrastant fortement avec les promesses antérieures de qualité "immaculée" et causant une migration croissante des utilisateurs vers OpenAI ou Codex malgré les réserves éthiques.
