## Résumé de l'article

OpenAI a lancé GPT-5.5, un modèle frontier pour les tâches professionnelles complexes, ainsi qu'une variante Pro optimisée pour les calculs intensifs. Ces nouveaux modèles supportent une fenêtre de contexte de 1M tokens, l'entrée d'images, les sorties structurées et plusieurs autres capacités avancées.

- GPT-5.5 inclut par défaut un effort de raisonnement moyen et fonctionne uniquement avec le prompt caching étendu
- GPT Image 2, un modèle générative état-de-l'art, offre des tailles d'image flexibles et une tarification basée sur les tokens avec 50% de réduction sur l'API Batch
- Le SDK Agents a été mis à jour avec des bacs à sable contrôlés, l'inspection du harness open-source et une gestion personnalisée des mémoires
- GPT-5.4 mini et nano ont été lancés en mars pour les charges de travail à haut volume, avec des capacités réduites pour réduire les coûts
- L'API Sora s'est étendue avec des références de caractères réutilisables, des générations jusqu'à 20 secondes et une sortie 1080p à 0,70 $ par seconde

## Discussion sur Hacker News (55 commentaires analysés)

**Avis positifs** :
- Le modèle offre des gains réels en cohérence sur les tâches longues et en suivi des spécifications détaillées, notamment pour le code production et les workflows complexes, justifiant son usage premium pour les cas haute valeur.
- La disponibilité via API répond à des besoins réels en traduction médicale, support multilingue et accès à des services pour populations sans alternatives (urgences, pays sans ressources), même imparfaitement.
- Les utilisateurs expérimentés avec bonnes pratiques de prompting et orchestration multi-modèles rapportent une qualité de code comparable à du travail manuel très soigné, contredisant les critiques de "slop".

**Avis négatifs** :
- Doublage des coûts par rapport à GPT-5.4 sans amélioration de performance notable ; les modèles concurrents (Opus, Sonnet) offrent meilleur rapport qualité-prix, suggérant une stratégie d'extraction de valeur plutôt que d'innovation.
- Le modèle reste paresseux (génère du code incomplet, refuse les requêtes justes avec "I can't do that"), produit des hallucinations, des oublis de code et une verbosité excessive sur la documentation, sans progrès significatif sur les trois dernières générations.
- Les safeguards et filtres éthiques sont contreproductifs et décalés des vrais besoins du monde réel ; OpenAI optimise pour la responsabilité légale plutôt que l'accessibilité réelle, blâmant les utilisateurs pour des refus qui déplacent simplement le risque.
