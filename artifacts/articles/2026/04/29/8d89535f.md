## Résumé de l'article

À partir du 1er juin 2026, les révisions de code effectuées par GitHub Copilot consommeront les minutes GitHub Actions en plus des crédits IA. Ce changement s'applique à tous les plans Copilot (Pro, Pro+, Business, Enterprise) pour les dépôts privés, tandis que les dépôts publics restent gratuits.

- Les révisions Copilot seront facturées selon deux mécanismes : crédits IA pour l'utilisation de Copilot, et minutes GitHub Actions consommées depuis votre allocation existante pour les dépôts privés
- L'architecture agent-powered de Copilot code review s'exécute sur des runners GitHub-hosted (avec options self-hosted et larger runners aux tarifs différents)
- Les utilisateurs doivent examiner leur consommation GitHub Actions actuelle, vérifier les limites de budget, et partager cette information avec leurs administrateurs avant le 1er juin 2026
- Aucune configuration supplémentaire n'est requise si vous utilisez déjà des GitHub-hosted Runners
- Jusqu'au 1er juin 2026, l'utilisation de Copilot code review continue de ne consommer que les unités de demande premium (PRU) Copilot

## Discussion sur Hacker News (182 commentaires)

**Avis positifs** :
- GitHub Copilot Code Review s'exécute sur Actions, donc facturer à partir du budget Actions est cohérent avec l'infrastructure réelle utilisée
- Les prix des API d'inférence chez les principaux fournisseurs suggèrent que la facturation au coût réel est justifiée et probablement rentable
- C'est une étape nécessaire : les entreprises doivent arrêter de subventionner les services d'IA avec l'argent des investisseurs et facturer le coût véritable
- La facturation transparente par utilisation (tokens + minutes) reflète mieux les coûts réels que les abonnements plafonnés illimités

**Avis négatifs** :
- C'est un double factage intentionnel qui crée de l'opacité et rend difficile de tracer les coûts réels (minutes Actions + crédits IA)
- GitHub a brûlé sa bonne volonté accumulée en abandonnant progressivement les tarifs bas sans préavis : c'est un classique 'rug pull' de startup bien financée
- Les augmentations de prix (27x plus chères selon certains utilisateurs) forcent les entreprises à migrer vers des alternatives ou des modèles locaux, cédant le marché aux fournisseurs étrangers moins chers
- Les Services GitHub Actions lui-même est peu fiable (98% d'uptime réel) avec des interruptions fréquentes, ce qui rend risqué de dépendre d'une facturation supplémentaire
- Copilot reste le plus faible des assistants de code sur le marché (inférieur à Cursor, Windsurf, Claude), donc augmenter les prix tout en dégradant la qualité garantit la perte de clients
