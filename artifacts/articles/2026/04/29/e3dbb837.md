## Résumé de l'article

Claude Code, l'interface d'Anthropic pour les développeurs, présente un bug où la présence de la chaîne sensible à la casse « HERMES.md » dans l'historique récent des commits git provoque le routage des requêtes API vers une facturation « usage supplémentaire » au lieu du quota inclus dans l'abonnement Max. Un utilisateur a ainsi consommé 200 dollars d'usage supplémentaire alors que son plan mensuel disposait de 86 % de capacité restante.

- Le bug est déclenché uniquement par la chaîne exacte « HERMES.md » dans les messages de commit git, pas par la présence d'un fichier portant ce nom sur le disque
- Claude Code inclut les commits récents dans le système prompt envoyé à l'API, et un mécanisme serveur routerait différemment les requêtes contenant cette chaîne
- La reproduction est triviale : un simple commit avec le message « add HERMES.md » suffit à provoquer l'erreur « out of extra usage »
- Anthropic refuse actuellement de rembourser les 200 dollars facturés à titre d'usage supplémentaire
- Le message d'erreur ne donne aucune indication sur le routage basé sur le contenu, rendant le diagnostic extrêmement difficile pour les utilisateurs affectés

## Discussion sur Hacker News (168 commentaires)

**Avis positifs** :
- La politique de refus de remboursement d'Anthropic pour erreurs techniques est juridiquement intenable dans plusieurs pays et devrait être contestée par des recours légaux ou des chargebacks bancaires
- Les pratiques de facturation abusive d'Anthropic semblent systémiques et récurrentes, affectant plusieurs clients, ce qui justifie une action collective ou régulatrice
- Le service client d'Anthropic utilisant des IA sans supervision humaine produit des réponses impersonnelles et contredictoires qui aggravent la situation et endommagent la réputation de l'entreprise
- Anthropic brûle rapidement la bonne volonté de sa base de développeurs par une succession de débâcles de facturation et des comportements anti-concurrentiels, créant une opportunité pour les concurrents

**Avis négatifs** :
- La distinction entre remboursement (de l'argent surcoûté) et compensation (pour préjudice/inconvénient) pourrait justifier une réponse partielle d'Anthropic qui aurait mal communiqué sa position
- GitHub n'est peut-être pas le canal approprié pour les réclamations de facturation, ce qui pourrait expliquer le traitement inadéquat; des canaux de support dédiés existent
- Les recours juridiques (small claims court) sont coûteux en temps et incertains dans leur exécution; les chargebacks risquent des représailles (fermeture de compte) sans bénéfice net pour le client
- Certains clients comme Anthropic reconnaissent le bug et ont corrigé le problème, suggérant que ce n'est pas une politique délibérée mais une défaillance de processus qui pourrait être rectifiée
