## Résumé de l'article

DeepSeek propose une API compatible avec les formats OpenAI et Anthropic, permettant d'utiliser les SDK existants pour accéder à ses modèles. Les anciens noms de modèles (deepseek-chat et deepseek-reasoner) seront dépréciés le 24 juillet 2026 au profit de deepseek-v4-flash.

- L'API DeepSeek utilise le format OpenAI/Anthropic, facilitant l'intégration avec les outils existants
- Après authentification avec une clé API, vous pouvez faire des appels via curl, Python (SDK OpenAI) ou Node.js
- Les paramètres clés incluent le choix du modèle (deepseek-v4-pro), l'activation du mode réflexif (thinking) et le niveau d'effort de raisonnement (reasoning_effort)
- Les réponses peuvent être en mode stream ou non selon le paramètre stream configuré

## Discussion sur Hacker News (82 commentaires analysés)

**Avis positifs** :
- Le modèle atteint un niveau frontier comparable à Opus 4.6, avec des performances de pointe sur plusieurs benchmarks, à un prix révolutionnaire ($0.14-$3.48/M tokens contre $2+/M pour Claude).
- DeepSeek confirme sa position de leader en open-source : publication systématique des weights, papers et frameworks (TileKernels, DeepGEMM, etc.), contrairement aux grands labs occidentaux qui verrouillent leurs modèles.
- L'architecture innovante (manifold-constrained hyper-connections, attention hybride) permet une efficacité d'entraînement remarquable ; la version Flash (284B, 13B actifs) devient accessible sur matériel local classique.
- La documentation technique est exceptionnellement claire et complète, surpassant celle d'OpenAI/Google en concision et utilité pratique.
- Le rapport qualité/prix déverrouille de nouveaux usages : agents, applications coûteuses deviennent viables ; les prix suggèrent une rentabilité réelle, pas une subvention intenable.

**Avis négatifs** :
- Les comparaisons de benchmarks sont suspectes : l'interne de DeepSeek reconnaît que v4 est inférieur à Opus 4.6 Thinking ; les gains affichés semblent partiellement liés au benchmark-maxing plutôt qu'à une véritable supériorité.
- L'absence d'interface native (pas de "DeepSeek Code") crée une friction majeure pour les développeurs habitués à Claude ; l'intégration reste technique et demande configuration manuelle.
- Les préoccupations géopolitiques et de confidentialité freinent l'adoption : envoi de données PII vers des serveurs en Chine, convergence implicite avec l'État chinois soulèvent des réserves légitimes chez les entreprises US.
- Le cycle hype-oubli se répète à chaque sortie DeepSeek ; les avantages réels finissent marginalisés face à la consolidation autour des écosystèmes Anthropic/OpenAI malgré le coût supérieur.
- Exécution limitée : l'intégration avec des outils agents (Pi Coding) révèle des défauts fonctionnels immédiatement après le lancement.
