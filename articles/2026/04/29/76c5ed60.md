## Discussion sur Hacker News (319 commentaires)

**Avis positifs** :
- La clarification progressive du droit d'auteur est nécessaire : les autorités de régulation (Office américain du droit d'auteur, cours) établissent progressivement qu'une contribution humaine significative est requise pour obtenir une protection copyright sur le code généré par IA.
- L'analogie avec les outils existants est pertinente : un compilateur, un IDE ou Photoshop ne confèrent pas de droits à leurs éditeurs sur le code ou les images produits ; Claude Code devrait être traité identiquement comme un outil mathématique sophistiqué.
- L'enjeu réel dépasse le copyright : pour les entreprises qui adoptent massivement les outils d'IA, les vrais problèmes sont la contamination par les licences copyleft, l'inceritde en due diligence lors de fusions-acquisitions, et la responsabilité en cas de code malveillant ou contrefait.
- La documentation des choix humains est cruciale : maintenir un historique détaillé des prompts, rejets de solutions IA et modifications humaines crée une trace légale de la contribution créative et devient une défense potentielle.
- Le droit d'auteur traditionnel s'applique à la sortie : si le code généré reproduit substantiellement du code copyrighted dans les données d'entraînement, les détenteurs de droits d'auteur originaux conservent leurs recours, indépendamment de qui possède nominalement la sortie.

**Avis négatifs** :
- Le consensus juridique sur la contamination par copyleft n'est pas établi : l'article prétend un consensus émergent, mais aucune cour n'a tranché définitivement sur la responsabilité des développeurs utilisant du code contaminé par des données d'entraînement dont la provenance exacte est invisible.
- Les contrats de travail et l'indemnisation par Anthropic créent une fausse certitude : les entreprises se comportent comme si elles possèdent le code généré en vertu de clauses d'assignment IP, mais cette confiance repose sur un pari légal non résolu, pas sur un droit établi.
- Le refus de cert par la Cour suprême ne règle rien : contrairement à ce que suggère l'article, le déni de pourvoi en Thaler n'établit pas le droit national ; seule la décision du circuit DC reste en vigueur dans cette juridiction, et d'autres circuits pourraient différer.
- La distinction « transformatif vs déterministe » est exagérée : les chercheurs font des progrès vers des LLM déterministes (température 0), ce qui pourrait changer radicalement la classification légale du code généré, contrairement aux images d'IA jugées non-copyrightables.
- La réalité pratique diverge de la théorie légale : en 25 ans, pratiquement aucun contentieux pour violation de copyleft n'a abouti, et les développeurs réutilisent massivement du code sans attribution ; l'ajout de l'IA ne changera probablement rien à l'impuissance des détenteurs de droits mineurs.
