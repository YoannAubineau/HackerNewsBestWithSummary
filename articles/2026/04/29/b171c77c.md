## Résumé de l'article

Zed, un éditeur de code construit à partir de zéro en Rust avec une architecture basée sur les GPU plutôt que sur Electron, annonce sa version 1.0 après cinq années de développement. L'application intègre nativement l'IA avec support multi-agents et propose une expérience collaborative homme-IA pour le codage.

- Zed a été réinventé sans dépendre de Chromium ou Electron en utilisant un framework UI personnalisé (GPUI) et en organisant l'application autour des shaders GPU, similaire à un moteur de jeu vidéo
- L'éditeur supporte plus d'une douzaine de langages, Git, SSH, le débogage et autres outils essentiels sur Mac, Windows et Linux (>1 million de lignes de code)
- L'IA est intégrée au cœur plutôt que greffer en surface, avec support du protocole Agent Client pour Claude, Codex, OpenCode et autres agents
- Une version Zed for Business est lancée pour les entreprises avec facturation centralisée, contrôle d'accès par rôles et gestion d'équipe
- DeltaDB, un moteur de synchronisation basé sur les CRDT, est en développement pour permettre la collaboration en temps réel entre humains et agents sur le même code

## Discussion sur Hacker News (177 commentaires)

**Avis positifs** :
- Performance et réactivité supérieures à VS Code, particulièrement sur les gros projets; consommation mémoire inférieure
- Intégration Claude/IA bien intégrée au sein de l'éditeur avec une vue du code lors des conversations, meilleure expérience que le terminal seul
- Expérience utilisateur fluide et épurée, fonctionnalités bien configurées par défaut (LSP, Tree-sitter), facilité d'utilisation immédiate
- Édition à distance via SSH et conteneurs de développement fonctionnels, parallélisant bien avec les workflows modernes
- Qualité du logiciel, stabilité, amélioration constante à chaque release avec une équipe réactive aux retours

**Avis négatifs** :
- Nombreuses fonctionnalités manquantes par rapport à VS Code/JetBrains: refactoring avancé, hiérarchie d'appels, gestion des sous-agents, affichage des images, support des notebooks Python
- Expérience de recherche problématique: ouverture d'onglets persistants au lieu de modaux, validation requise, UX incomparable aux éditeurs classiques (Sublime, Telescope)
- Support de langages incomplet: intégrateur LSP par défaut déclenche des avertissements excessifs sur code legacy, configuration peu intuitive pour utilisateurs novices
- Problèmes Linux persistants: raccourcis sur claviers non-latins infonctionnels, couleurs incorrectes en Wayland, police bitmap non supportée malgré des issues ouvertes depuis 2 ans
- Interface graphique et thèmes par défaut très ternes avec peu de contraste, navigation confuse avec des icônes trop petites sur écrans haute résolution
