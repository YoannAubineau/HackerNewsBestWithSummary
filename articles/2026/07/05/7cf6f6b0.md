## Résumé de l'article

Shadcn/ui, une collection de composants headless non stylisés construits sur des bibliothèques existantes, fait passer Base UI en tant que choix par défaut pour les nouveaux projets, remplaçant Radix qui équipait la plateforme depuis son lancement en janvier 2023. Base UI, construite par les mêmes auteurs que Radix avec l'expérience acquise, a atteint la stabilité (version 1.6.0, 6 millions+ téléchargements hebdomadaires) et est maintenant privilégiée par la communauté.

- Radix reste pleinement supporté et ne sera pas supprimé ; les projets existants n'ont pas besoin de migrer, tous les nouveaux composants seront livrés pour les deux bibliothèques.
- Les nouveaux projets créés avec `npx shadcn init` utiliseront désormais Base UI par défaut, bien qu'un drapeau `-b radix` permette de conserver Radix.
- Une compétence de migration progressive a été développée pour les projets Radix existants souhaitant passer à Base UI : elle migre composant par composant, préserve les personnalisations de code, génère un historique git propre avec un commit par composant, et peut être supervisée par des agents d'IA (Claude, Cursor).
- Les rapports de migration détaillent les changements par composant, signalent les comportements modifiés sans les appliquer silencieusement, et un test réel sur 60+ composants s'est déroulé en environ 25 minutes.

## Discussion sur Hacker News (97 commentaires)

**Avis positifs** :
- Base UI offre des primitives plus bas-niveau et moins dogmatiques que Radix, permettant une meilleure composition et un contrôle accru pour les cas d'usage avancés tout en restant simple au niveau shadcn
- L'approche copy-paste de shadcn évite les problèmes de dépendances liées : chaque composant peut évoluer indépendamment sans bloquer les mises à jour globales, contrairement aux bibliothèques traditionnelles
- La qualité globale de shadcn est largement reconnue, notamment pour la construction d'applications desktop avec un design épuré et professionnel
- Le passage à Base UI ouvre potentiellement la voie à supporter d'autres frameworks au-delà de React, contrairement au lock-in de Radix

**Avis négatifs** :
- La plupart des sites restent encore sur la version Radix, ce qui suggère que ce changement risque de fragmenter l'écosystème et la documentation existante
- Le recours à des agents IA pour migrer le code est un symptôme alarmant du churn technologique permanent dans le frontend, alors qu'une simple mise à jour de version devrait suffire
- L'approche copy-paste crée des problèmes de maintenance : il faut vérifier chaque ligne de CSS personnalisée à chaque upgrade, transformant les mises à jour en projet majeur
- Il n'y a pas d'explication claire sur les avantages concrets de Base UI par rapport à Radix au-delà de metrics d'adoption, et la communauté continue de migrer sans comprendre le 'pourquoi'
- Shadcn reste limité à React alors que d'autres solutions offrent un support multi-frameworks (Skeleton pour Solid.js, ports Svelte/Vue), réduisant son attrait pour l'écosystème JavaScript diversifié
