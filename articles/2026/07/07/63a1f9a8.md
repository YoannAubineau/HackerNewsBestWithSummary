## Discussion sur Hacker News (55 commentaires)

**Avis positifs** :
- Le projet démontre une excellente exécution technique : quantization-aware training, inférence WASM SIMD optimisée, et transparence complète du code d'entraînement; félicitations pour le partage du pipeline.
- Les cas d'usage concrets sont validés : recherche sémantique en temps réel, appariement d'intentions FAQ, clustering, et recherche local-d'abord sur corpus privés (Django docs confirmé fonctionnel).
- L'approche on-device élimine les dépendances API backend et offre une confidentialité supérieure, avec des performances acceptables (2.5-5 ms/embedding) pour des applications réelles.
- Le modèle est significativement plus petit que les alternatives comparables (7 MB vs gte-small) avec fidélité teacher de 0.84 Spearman, offrant un bon compromis taille/qualité.
- Potentiel d'écosystème décentralisé : intégration avec recherche statique (HNSW sur Parquet, SQLite en HTTP range requests) pourrait créer des alternatives aux moteurs de recherche centralisés.

**Avis négatifs** :
- Performance réelle en WASM variable selon l'architecture (rapporté 35 emb/sec en Firefox i5-4570 vs 400/sec annoncés, possible fallback non-SIMD détecté).
- Limitations linguistiques : le modèle semble optimisé pour l'anglais; support multilangue non clairement documenté.
- Problèmes de compatibilité matérielle : plantages confirmés sur chips Apple M-series et A-series (iPhones), service worker instable en Safari.
- Préoccupations sur l'auto-exécution de code WASM dans le navigateur sans consentement explicite : risques potentiels similaires aux mineurs de crypto, bien que la communauté relativise versus JavaScript existant.
- L'indexation initiale de 30 secondes peut être lourde pour l'utilisateur; alternative backend suggérée mais non intégrée par défaut.
