## Résumé de l'article

Steve Schultze, fondateur de Val Town, affirme que l'apprentissage du code conserve sa valeur éducative et intellectuelle, bien que la promesse d'un enrichissement rapide ait disparu. Il place cet apprentissage au même niveau que les arts libéraux : un développement personnel et cognitif plutôt qu'une simple voie professionnelle.

- L'idée que « apprendre à coder » garantit une carrière lucrative est dépassée, contrairement aux promesses des années précédentes
- La programmation enseigne des méta-compétences transversales : déboguer, composer, raisonner logiquement, et surtout découvrir qu'on peut apprendre n'importe quoi
- Coder combine créativité (comme l'écriture), précision (comme les mathématiques) et boucles de rétroaction instantanées (comme les jeux vidéo), ce qui en fait une activité intellectuelle riche
- L'émergence des LLM n'invalide pas l'apprentissage du code, tout comme les LLM ne menacent pas la pertinence des humanités
- Le code est le fondement du fonctionnement du monde moderne ; maîtriser son langage formel, c'est acquérir un pouvoir réel de transformation

## Discussion sur Hacker News (217 commentaires)

**Avis positifs** :
- Apprendre à coder enseigne la pensée logique et structurée, utile bien au-delà de la programmation : résolution de problèmes, décomposition en étapes, itération et débogage sont des compétences transférables à de nombreux domaines.
- Comprendre les fondamentaux (architectures, abstractions, design) reste essentiel pour bien utiliser les LLM et discerner une bonne solution d'une mauvaise, même en ère d'IA ; les LLM amplifient les capacités des programmeurs compétents, pas des novices.
- La programmation est un acte créatif et expressif qui offre une satisfaction intrinsèque, au-delà de toute considération économique ou d'employabilité ; c'est un art de formaliser des processus.
- Les domaines critiques (compilateurs, frameworks, outils fondamentaux) nécessitent toujours une maîtrise profonde du code ; les zones de tolérance zéro exigent des développeurs compétents, pas du code généré.
- La capacité à conceptualiser, raffiner les abstractions et construire des systèmes complexes et maintenables reste un atout professionnel durable, particulièrement pour les ingénieurs seniors et les architectes techniques.

**Avis négatifs** :
- Le marché du travail se contracte : même une amélioration modérée de productivité (30%) réduit la demande de développeurs, et la suroffre actuelle de compétences rend l'apprentissage du code économiquement peu attrayant pour les nouveaux entrants.
- Le seuil requis pour « suffisant » a baissé ; les LLM rendent l'apprentissage manuel du code moins nécessaire pour produire du code fonctionnel, et les managers ne distinguent souvent plus entre du bon code et du code vibe-généré.
- L'enseignement du code a été survendu comme panacée universelle (remédier au chômage, s'enrichir rapidement) ; or c'est un métier comme les autres avec des exigences spécifiques, pas une garantie de prospérité.
- Le risque de boucle négative : si tout le code du futur vient de LLM entraînés sur du code humain, les modèles dégénéreront progressivement et produiront du code de qualité décroissante, non maintenable et non compréhensible.
- Pour les débutants, apprendre à coder face à la concurrence des veterans (qui maîtrisent déjà les LLM) et des outils qui réduisent les tâches de base est devenu extrêmement difficile et peu rentable en termes d'insertion professionnelle.
