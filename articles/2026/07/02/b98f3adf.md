## Résumé de l'article

ZCode est un outil d'intégration profonde avec GLM-5.2, le modèle de codage d'intelligence artificielle. Le plan « GLM Coding Lite » offre un accès abordable aux outils de codage assisté.

- Plan Lite proposé à 16,2 $/mois (prix réduit de 18 $) avec allocation d'usage de base incluse
- Conçu pour les itérations légères sur petits référentiels
- Accès continu aux derniers modèles phares et fonctionnalités
- Support de plus de 20 outils de codage, dont l'intégration profonde de ZCode

## Discussion sur Hacker News (253 commentaires)

**Avis positifs** :
- GLM-5.2 démontre des capacités réelles et comparables à Claude Opus pour certaines tâches, notamment en refusant moins souvent les demandes de sécurité
- L'intégration avec les fournisseurs CLI existants (OpenCode, Pi) permet aux utilisateurs de basculer facilement entre harnesses sans dépendre d'un éditeur propriétaire
- Le support Linux dès le lancement et la possibilité de se connecter à des conteneurs Docker ou VMs SSH offrent de la flexibilité pour les utilisateurs avancés
- La tarification avec quota réduit pour GLM-5.2 pendant les heures creuses (0.67x hors-pic) offre une meilleure économie que les concurrents pour les utilisateurs réguliers

**Avis négatifs** :
- L'application est fermée (non open-source) malgré que les poids GLM soient ouvertes, soulevant des inquiétudes sur la surveillance de données et la confiance envers les fournisseurs chinois
- GLM-5.2 est nettement plus lent qu'Opus 4.8 (environ deux fois plus lent en moyenne), ce qui affecte la productivité pour les tâches critiques
- L'interface utilisateur souffre de bogues (correcteur orthographique sans suggestions, crashes fréquents, messages d'erreur répétés 'Cannot connect to API: write EPIPE')
- La proposition de valeur est limitée—c'est essentiellement une copie de Codex/Claude Code sans innovation significative, avec les plans tarifaires calqués sur Anthropic
