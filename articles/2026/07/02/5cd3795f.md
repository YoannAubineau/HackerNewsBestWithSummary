## Résumé de l'article

Kimi K2.7 Code est un modèle d'IA open-weight (poids ouvert) proposé pour la première fois comme option sélectionnable dans GitHub Copilot, offrant une alternative moins coûteuse aux modèles existants. Le modèle est hébergé par GitHub sur Microsoft Azure et facturé selon les tarifs du fournisseur.

- Le déploiement débute progressivement sur les plans Copilot Pro, Pro+, et Max, avec expansion prévue aux plans Business et Enterprise dans les prochaines semaines
- Le modèle est sélectionnable via le sélecteur de modèles dans Visual Studio Code, Visual Studio, JetBrains, Xcode, Eclipse, Copilot CLI, et sur github.com
- Pour Copilot Business et Enterprise, le modèle est désactivé par défaut ; les administrateurs doivent activer la politique correspondante dans les paramètres
- Les administrateurs sont invités à évaluer le modèle open-weight selon leurs exigences de sécurité, conformité et gouvernance des données avant de l'activer

## Discussion sur Hacker News (147 commentaires)

**Avis positifs** :
- Kimi K2.7-Code offre une alternative compétitive aux grands modèles propriétaires, avec des performances comparables à Claude Sonnet 4.6 selon les benchmarks, et s'avère significativement moins cher à l'usage que GPT-5.4 mini ou d'autres solutions premium.
- GitHub Copilot dispose désormais d'une position de marché solide pour l'entreprise : support de multiples modèles (Claude, GPT, Gemini, Kimi), CLI robuste, agents hébergés, et workflows d'IA dans CI, offrant une flexibilité supérieure aux concurrents.
- L'arrivée de modèles open-weight dans Copilot et les solutions alternatives (OpenRouter, Bedrock, OpenCode, Deepseek) diversifie les options et crée une réelle concurrence qui bénéficie aux utilisateurs, notamment pour les budgets contraints des entreprises.
- Les modèles locaux (Qwen 3.6, GLM 4.7) deviennent viables pour les développeurs : stabilité prévisible, absence de changements de prix impromptus, et capacités suffisantes pour de nombreux usages, même sur du matériel modeste.
- La compétition s'intensifie dans les modèles de code, ce qui force l'innovation : le choix n'est plus binaire mais stratégique selon la tâche, le coût et le contexte d'utilisation.

**Avis négatifs** :
- Le changement de tarification de GitHub Copilot en juin 2024 est considéré comme désastreux : les coûts ont été multipliés par 3 à 6x, épuisant les budgets mensuels en quelques jours et forçant de nombreux utilisateurs et départements entiers à migrer vers Claude Code ou des alternatives.
- La harnache (système prompt et intégration) de Copilot est régulièrement décrite comme inférieure à celle de Claude ou d'autres solutions, causant une sous-performance de 3 à 4 fois sur 5 même avec le même modèle Claude, ce qui questionne la valeur réelle de la plateforme.
- Les benchmarks de Kimi K2.7 sont remis en question : plusieurs utilisateurs rapportent que le modèle 'tourne en rond' sur des tâches que Claude résout directement, et les performances dépendent fortement de si la version servie est quantifiée ou complète.
- Copilot a perdu la confiance des utilisateurs en raison d'une stratégie produit peu fiable : prix explosifs, nerfing subjectif de capacités, disparition de features, et documentation obsolète, tandis que les alternatives (Claude Code, Cursor) offrent désormais une meilleure expérience.
- L'inférence en nuage (Azure) est soumis au Cloud Act américain et n'offre aucune conformité GDPR garantie, posant des risques de confidentialité pour les entreprises européennes malgré le verrouillage du marché par les cloud providers majeurs.
