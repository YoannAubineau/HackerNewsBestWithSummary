## Résumé de l'article

Reddit a annoncé que old.reddit.com (l'ancienne interface) nécessite désormais une connexion pour être consultée, justifiant cette décision par des problèmes de sécurité et de scraping. Un utilisateur analyse cette mesure et conclut que la raison invoquée est prétexte : old Reddit sert simplement du HTML brut, contrairement à new Reddit qui charge beaucoup plus de code JavaScript et est techniquement plus complexe.

- Reddit affirme que old.reddit.com est une source majeure de « scraping abusif », mais new Reddit reste accessible sans connexion et semble tout aussi scrappable
- Old Reddit est une interface légère (1 MB de données) qui fonctionne sans JavaScript ; new Reddit consomme 5 fois plus et exige JavaScript pour charger le contenu
- L'ajout de JavaScript et de requêtes tracking dans new Reddit ne constitue pas une « sécurité » mais simplement un moyen de rendre le scraping plus difficile
- L'auteur soupçonne que Reddit veut plutôt protéger l'accès aux données utilisateur (considérées comme précieuses pour les LLM) que résoudre des problèmes de sécurité
- La mesure affecte les utilisateurs marginaux (ceux utilisant Firefox, noscript, des interfaces anciennes) et représente selon l'auteur une évolution négative de l'internet décentralisé

## Discussion sur Hacker News (285 commentaires)

**Avis positifs** :
- Les scrapers posent un problème réel et coûteux pour les sites complexes ; limiter l'accès non authentifié réduit le traffic de scrapers qui surcharge les serveurs
- La sécurité requiert des mécanismes modernes (détection de bots, captchas, javascript) impossibles à implémenter en HTML pur ; c'est une évolution normale du cycle de vie logiciel
- Reddit a des accords de licencing lucratifs avec OpenAI et Google ; protéger les données justifie les restrictions pour contrôler qui accède au contenu
- Exiger une connexion réduit l'automatisation des postes par des bots, améliorant théoriquement la qualité des discussions humaines

**Avis négatifs** :
- Reddit invoque la sécurité comme prétexte transparent pour monétiser ; la vraie raison est de forcer les utilisateurs vers la nouvelle interface riche en publicités et données de suivi
- HTML pur est facile à cacher des bots via JavaScript sans supprimer l'ancien interface ; Reddit peut bloquer les scrapers sans nuire aux utilisateurs légitimes
- Exiger une connexion crée une friction majeure pour les lecteurs occasionnels, détériorant l'UX et violant le principe que l'anonymat sur internet doit rester accessible
- Reddit a autoprovoqué la crise des scrapers en fermant l'API publique et en tentant de contrôler les données exclusivement ; avant, Pushshift et les API officielles régulaient l'accès
- Le contenu Reddit s'est dégradé (bots, astroturfing, censure modérée) ; forcer les gens à se connecter aggrave le problème plutôt que de le résoudre
