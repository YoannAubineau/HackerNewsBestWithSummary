## Résumé de l'article

Cet article explique comment utiliser un hook MessageDisplay dans Claude pour remplacer automatiquement certains termes ou phrases par d'autres selon vos préférences. Un script Python utilise des expressions régulières pour intercepter et modifier le texte des réponses de Claude avant affichage.

- Un script Python stocké dans `~/.claude/hooks/wordswap.sh` lit les messages de Claude et applique des remplacements de texte configurables
- Le script charge les remplacements depuis un dictionnaire (ex. : remplacer « load-bearing » par « cooked ») et utilise des regex insensibles à la casse
- La configuration s'ajoute au fichier `~/.claude/settings.json` dans le bloc `hooks` en spécifiant le type de hook `MessageDisplay`
- Le script doit être rendu exécutable avec `chmod +x` et les modifications prennent effet au démarrage d'une nouvelle session
- Les utilisateurs peuvent adapter les remplacements selon leurs besoins, de manière humoristique ou pour corriger des termes répétitifs

## Discussion sur Hacker News (335 commentaires)

**Avis positifs** :
- Les phrases répétitives de Claude (load-bearing, belt-and-suspenders, smoking gun) sont des termes utiles et précis qui capturent efficacement des concepts techniques communs, particulièrement en architecture et ingénierie
- La solution proposée via regex/hooks de filtrage est pratique, économe en tokens et ne compromet pas les capacités cognitives du modèle contrairement à modifier les poids d'entraînement
- Plusieurs commentaires reconnaissent que certains de ces termes existaient bien avant les LLM et que leur suremploi relève surtout d'un phénomène d'amplification massive plutôt que d'une invention du modèle
- Le problème n'est pas tant le vocabulaire en lui-même que la répétition excessive et prévisible ; les humains développent aussi leurs propres tics verbaux, mais sans cette uniformité massifiée

**Avis négatifs** :
- L'omniprésence de ces tics linguistiques chez Claude et autres LLM crée une monotonie oppressante et une dégradation cognitive pour les lecteurs ; des milliardsde tokens générés quotidiens avec le même style rend la détection triviale et fatigante
- Ces formulations apparemment 'intelligentes' masquent souvent un manque de précision réelle : Claude utilise 'load-bearing' pour des choses simplement 'utiles' et non véritablement critiques, transformant le jargon en superficialité
- La formation par RLHF amplify artificiellement ces tics dans une boucle de rétroaction où les outputs générés réutilisés en SFT renforcent les patterns ; aucune solution d'ajustement post-entraînement ne peut vraiment résoudre le problème structural
- Claude imite maintenant davantage le style de GPT et autres modèles (surtout depuis Opus 4.7), perdant sa voix distincte antérieure ; les humains commencent aussi à reproduire ces formulations, propageant un 'slop' linguistique contaminer le corpus textuel global
