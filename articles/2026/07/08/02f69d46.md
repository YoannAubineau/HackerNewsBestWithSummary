## Résumé de l'article

Uniqlo et Akamai ont commercialisé un t-shirt contenant un script bash obfusqué encodé en base64 comme easter egg pour leur campagne « Peace for All ». Le script, une fois décodé et exécuté, affiche une animation en boucle du message « PEACE FOR ALL » qui ondule horizontalement sur le terminal avec un dégradé de couleurs cyan à orange.

- Le script bash est imprimé en clair au dos du t-shirt avec un shebang (#!/bin/bash), y compris la chaîne base64 complète prête à être décodée
- L'auteur a utilisé l'OCR (circle-to-search Android, Tesseract, Claude) pour transcrire le code depuis le textile physique, processus fastidieux sans correction d'erreur
- Le script décodé contient des messages de félicitation en anglais et en japonais, puis une boucle infinie animant chaque caractère du texte selon une courbe sinusoïdale
- Le design du t-shirt référence l'histoire d'Internet : la couleur beige représente les boîtiers plastiques des ordinateurs des débuts, le cœur symbolise le bien-fondé d'Internet, et le code Linux en arrière-plan célèbre le langage open-source qui unit Akamai et ses partenaires
- Le t-shirt utilise la police Consolas (plutôt associée à Windows), une choix typographique notable pour du contenu orienté vers Linux et le bash

## Discussion sur Hacker News (81 commentaires)

**Avis positifs** :
- Le script est clairement une création humaine intentionnelle, conçu délibérément pour être difficile à OCRiser selon la vidéo du designer, ce qui explique les choix de design apparemment étranges
- L'article démontre efficacement comment les outils OCR modernes (Claude, Gemini) peuvent reconstruire le code en une seule tentative malgré les erreurs de transcription, montrant les progrès de la vision par IA
- C'est une excellente campagne marketing et collaborations intelligentes entre Uniqlo et Akamai, créant un engouement réel autour de l'objet à travers ce défi communautaire
- Le parallèle avec les t-shirts DeCSS historiques montre comment les vêtements deviennent des vecteurs d'expression technique créative et subversive

**Avis négatifs** :
- Plusieurs indices techniques (structure de calculs bc redondants, absence de sleep défensive, traitement inefficace des couleurs xterm-256, echo -n suivi de echo plutôt que echo seul) suggèrent que le script décodé a probablement été généré par une IA, contredisant l'idée d'une création purement humaine
- L'argument selon lequel les commentaires nombreux seraient humains est faible : il existe des raisons pratiques (longueur pour le design, espacer les caractères surlignés) pour ajouter des commentaires même en code généré
- Le script a des faiblesses pratiques : il dépend de `bc` (absent sur certaines distributions comme Alpine), manque de `set -euo pipefail`, et les choix de gradient xterm-256 n'ont aucun sens technique puisque ces couleurs ne forment pas un vrai gradient
- La transcription manuelle aurait probablement été plus efficace que les multiples tentatives OCR, suggérant une approche sous-optimale face au problème
