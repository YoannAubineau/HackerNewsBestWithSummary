## Résumé de l'article

Odra est un service de refactorisation de code qui aide les entreprises à nettoyer et optimiser les codebases générées par l'IA, réduisant la duplication et améliorant la maintenabilité.

- Le service analyse gratuitement le projet et propose une réduction de lignes de code cible (par exemple passer de 100 000 à 35 000 lignes en conservant la fonctionnalité)
- Une équipe de trois ingénieurs seniors intervient sur une semaine et docummente d'abord précisément le comportement de l'application avant de refactoriser
- Le tarif fixe est de dix mille dollars, avec une facturation au prorata si la cible de réduction n'est partiellement atteinte (par exemple 4 000 dollars si 40 % du réduction promise est livrée)
- Le service inclut une garantie de deux semaines et fournit des guardrails (fichiers CLAUDE.md, règles lint, vérifications CI) pour ralentir l'accumulation de dette technique lors du développement futur
- Le comptage des lignes se fait avec l'outil scc et exclut les commentaires, avec une interdiction explicite du code golf pour conserver la lisibilité

## Discussion sur Hacker News (151 commentaires)

**Avis positifs** :
- La demande est réelle : le volume de code AI généré explose (commits multipliés par 15x en un an), créant un vrai marché de nettoyage et de refactorisation
- C'est comparable à d'autres cycles historiques : tout comme les migrations cloud, les intégrations crypto ou l'externalisation, il y a toujours eu besoin de nettoyer après les tendances technologiques (les humains ont écrit du « slop » bien avant l'IA)
- Les ingénieurs expérimentés peuvent rapidement identifier et éliminer la complexité inutile (30% facile, 30% difficile, le reste nécessite vraie expertise) - c'est un travail que les juniors ou l'IA seule ne peuvent pas faire
- L'IA utilisée correctement par des développeurs compétents avec specs claires, revues de code strictes et tests robustes produit du code meilleur que le code humain traditionnel
- Le service comble une lacune : beaucoup de petites entreprises généreront du code AI en masse mais n'ont pas l'expertise interne pour l'assainir

**Avis négatifs** :
- Une semaine est insuffisante pour comprendre les règles métier complexes et les cas limites qui évoluent depuis des années dans les codebases réelles
- La garantie de deux semaines est insuffisante : les bugs subtils peuvent rester cachés bien plus longtemps, créant une fausse sécurité
- Utiliser l'IA pour nettoyer du code généré par l'IA revient à appliquer deux compressions avec perte successives où les erreurs se multiplient plutôt que s'annuler
- Le marketing et la rédaction du site ressemblent eux-mêmes à du contenu généré par l'IA, ce qui sape la confiance dans la capacité du service à vraiment améliorer la qualité
- C'est probablement une solution temporaire et illusoire : les clients re-vibecoderont par-dessus le code refactorisé, créant un cycle perpétuel ou simplement abandonnant le code pour partir de zéro
