## Résumé de l'article

Late.sh est une plateforme sociale accessible via SSH qui fonctionne dans le terminal, complétée par un client optionnel pour la voix, la musique et le partage multimédia. Elle s'adresse aux informaticiens et combine communication en temps réel, jeux, créativité collaborative et réseautage professionnel.

- Accès minimal via SSH seul; le client binaire optionnel ajoute voix, radio, visualiseur audio et partage de contenu
- Fonctionnalités : radio en direct, jeux (2048, sudoku, solitaire, multiplayer), tableau ASCII collaboratif avec historique, profils professionnels, chat en temps réel
- Authentification par clé SSH uniquement (pas de mot de passe ni OAuth)
- Conservation minimale des données : empreinte de clé publique stockée, pas d'adresses IP, pas d'analytique, messages et scores liés à la clé
- Possibilité d'utiliser une clé temporaire pour tester sans laisser de trace permanente

## Discussion sur Hacker News (100 commentaires)

**Avis positifs** :
- Concept innovant rappelant l'âge d'or des BBS et IRC, adapté à l'ère moderne avec SSH - plaît à beaucoup de développeurs nostalgiques de l'internet ancien
- Accessibilité technique : fonctionnement avec un simple client SSH sans installation obligatoire de binaires, ce qui réduit les barrières d'entrée
- Richesse fonctionnelle : intègre chat, jeux (Sudoku, Nanogrammes, Tetris), démos en direct et accès à des jeux texte (NetHack, DCSS), bien au-delà d'IRC traditionnel
- Identification par clé SSH innovante : élimine les mots de passe et OAuth, offrant une approche décentralisée cohérente avec la philosophie des développeurs
- Communauté enthousiaste : plusieurs utilisateurs rapportent une expérience positive et montrent le projet à leurs collègues avec enthousiasme

**Avis négatifs** :
- Contraste d'accessibilité critique : design volontairement sombre rend le texte difficilement lisible même pour les personnes ayant une bonne vision, problème amplifié sur e-ink et avec les lecteurs d'écran
- Copywriting peu clair en LLM : les descriptions concises et maladroites du fonctionnement (pas de comptes, pas d'OAuth) ne communiquent pas efficacement les avantages réels pour les utilisateurs
- Sécurité/privacy insuffisamment documentée : fuites potentielles de clés publiques SSH côté serveur non explicitement abordées à la première visite, nécessitant des commandes alternatives peu visibles
- Pattern UX discutable : installation par copier-coller de script bash sur la page d'accueil sans lien GitHub ni information sur les contributeurs, ressemble à une dark pattern
- Absence de consentement explicite : le nom d'utilisateur SSH par défaut devient l'identité publique sans accord préalable, problématique pour les utilisateurs professionnels
