## Résumé de l'article

Fable a développé un format de conteneur appelé splat4d pour encoder et diffuser des séquences vidéo 3D basées sur des gaussiennes. Le format utilise la quantification bornée en erreur pour garantir une précision déterministe de chaque pixel, permettant un streaming efficace via des requêtes HTTP Range sans logique serveur.

- Le fichier splat4d contient un en-tête avec les limites et les pas de quantification, une section statique avec les masques et valeurs de base, et des chunks GOP (Group of Pictures) avec les flux clés et delta organisés temporellement
- La quantification de style SZ/ZFP assure que chaque attribut reste dans une marge d'erreur choisie par l'utilisateur, éliminant la dérive temporelle et garantissant des valeurs bit-identiques entre les décodeurs Rust et JavaScript
- Le format est conçu pour fonctionner natif avec les stockages objets statiques (S3, GCS, R2) en utilisant uniquement des requêtes HTTP Range, sans fichiers manifeste ni conteneur vidéo
- Les fichiers sont déjà compressés en zstd, obtenant une compression environ 2,5× supérieure aux méthodes génériques lossless sur les séquences testées
- Une implémentation WebGPU/WebGL2 du rendu splat est fournie avec un outil Python (splat4d encode) pour convertir des séquences d'images en fichiers splat4d seekables

## Discussion sur Hacker News (58 commentaires)

**Avis positifs** :
- Le format est effectivement innovant : application de techniques de compression MP4/H265 établies aux gaussian splats 4D, représentant un vrai travail d'intégration
- Le code est open source (MIT) et accessible, avec une démo fonctionnelle qui fonctionne sur plusieurs navigateurs et appareils
- Les cas d'usage potentiels sont intéressants : vidéos de concerts, pièces de théâtre, sports ou espaces générés par IA où l'interactivité spatiale crée de nouvelles possibilités narratives
- Le projet démontre un potentiel réel pour réduire les fichiers de splats animés (7,4 MB pour 2 secondes de démo)
- La contribution technique plaît aux développeurs et le format pourrait inspirer des améliorations futures

**Avis négatifs** :
- Les prétentions à la "nouveauté" sont exagérées : toutes les techniques employées sont bien établies, et des projets similaires (4splat, Braindance VR) existent déjà
- Les performances annoncées (640 MB/s) sont mal explicitées et causent confusion entre débit d'encodage et taille réelle des fichiers
- Problèmes de compatibilité importants : WebGPU n'est pas universel (nécessite des drapeaux Chrome, ne fonctionne pas sur certains navigateurs/appareils iOS), WebGL serait plus compatible
- Conceptuellement problématique pour les films : la liberté de caméra détruit le cadrage, élément d'expression clé en cinématographie; les jeux vidéo feraient mieux cette fonction
- Pas de vrai bouton pour changer de scènes malgré les panneaux HTML, UX mobile peu conviviale
