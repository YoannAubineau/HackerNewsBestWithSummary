## Résumé de l'article

Un développeur critique les pratiques commerciales d'Anthropic, entreprise éditrice de Claude, un grand modèle de langage IA. L'auteur dénonce le verrouillage écosystémique, l'instabilité des API, les augmentations de prix et les restrictions imposées aux utilisateurs de Claude.

- L'API Claude est instable et exclusive à Anthropic, tandis que Claude Code (interface principale) enferme les utilisateurs dans l'écosystème propriétaire avec peu de concurrence et des bugs persistants depuis mois ou années
- Anthropic a modifié sa politique tarifaire en juin 2024 : les crédits d'agent SDK remplacent désormais les allocations d'abonnement pour les outils tiers, transformant les économies précédentes (15-30x par rapport aux API) en tarification API complète
- L'entreprise a facturé rétroactivement les utilisateurs qui dépassaient les limites sans notification préalable et a détecté les outils tiers pour augmenter les frais
- Les modèles open source (Qwen, GLM) deviennent compétitifs pour la programmation et permettent via des passerelles comme OpenRouter de choisir librement le fournisseur et d'optimiser les coûts
- L'auteur dénonce comme contraires à l'éthique le verrouillage client, les restrictions artificielles présentées comme des améliorations, les tests de tarification dynamique et les changements de conditions sans consentement

## Discussion sur Hacker News (148 commentaires)

**Avis positifs** :
- Anthropic restreint légitimement l'accès aux abonnements pour éviter les abus massifs de tokens (des utilisateurs brûlaient des centaines de milliards de tokens par jour dans les harnesses tiers)
- Les modèles alternatifs (GLM 5.2, DeepSeek, Qwen) atteignent une parité de performance suffisante à une fraction du coût, offrant une vraie liberté de choix aux utilisateurs
- La stabilité des modèles open-source et des API tiers comme OpenRouter offre une protection contre le lock-in et les changements arbitraires de politique
- OpenAI démontre qu'il est possible de laisser les utilisateurs employer les abonnements avec des harnesses tiers sans enshittification massive

**Avis négatifs** :
- Le modèle économique des startups d'IA brûle du VC en subventionnant les services sans vision claire de rentabilité, ce qui rend l'enshittification inévitable à terme
- La fiabilité catastrophique d'Anthropic (9000+ issues non résolues, uptime faible, freezes) contredit leurs promesses marketing sur les agents autonomes
- Les changements de politique constants (interdictions, « pauses », flanc-mous) créent une incertitude qui justifie l'attrition des utilisateurs, indépendamment de la qualité du modèle
- Même avec le meilleur modèle, le coût réel pour les hardcore users (overflow de tokens, dégradation silencieuse vers Opus 4.8) rend Anthropic non compétitif face aux alternatives diversifiées
