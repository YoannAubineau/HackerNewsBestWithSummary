## Résumé de l'article

Grok Build est un agent de codage IA en ligne de commande développé par SpaceXAI, fonctionnant comme une interface texte plein écran (TUI) capable de comprendre votre base de code, modifier des fichiers, exécuter des commandes shell et effectuer des recherches web. Le code source Rust est désormais ouvert sous licence Apache 2.0 et disponible sur repository public avec des binaires précompilés pour macOS, Linux et Windows.

- Grok Build s'exécute en mode interactif, sans interface (headless) pour l'automatisation, ou embarqué dans des éditeurs via le protocole ACP
- L'installation se fait via des scripts bash (macOS/Linux) ou PowerShell (Windows), ou en compilant depuis les sources avec Rust et protoc
- La compilation locale utilise `cargo run` ou `cargo build` pour générer le binaire `xai-grok-pager`
- Documentation complète disponible sur docs.x.ai/build/overview avec guides d'authentification, raccourcis clavier, configuration et serveurs MCP
- Le code tiers et les dépendances conservent leurs licences d'origine listées dans THIRD-PARTY-NOTICES et NOTICE

## Discussion sur Hacker News (196 commentaires)

**Avis positifs** :
- L'ouverture du code source facilite l'audit de sécurité et permet à la communauté de vérifier réellement ce que le logiciel fait avec les données utilisateur, contrairement à la version fermée précédente.
- Le modèle Grok 4.5 est reconnu comme performant, plus rapide que Claude Opus et offrant une bonne alternative avec une vitesse d'exécution impressionnante pour les tâches de codage.
- L'open source stimule l'innovation communautaire avec plusieurs forks et améliorations déjà en développement (versions avec confidentialité renforcée, support multi-fournisseurs, interfaces alternatives).
- C'est un geste tactique intelligent pour restaurer la réputation après le scandale de l'exfiltration de données, montrant une tentative de transparence.

**Avis négatifs** :
- Le scandale de l'exfiltration de données utilisateur (fichiers .env, clés SSH, code source entier) reste non résolu : aucun certificat indépendant ne prouve que les données ont été supprimées, érodant la confiance.
- La base de code est excessivement volumineuse (1,3 million de lignes de Rust pour une simple interface agent), suggérant du code généré par IA peu optimisé plutôt qu'un design intentionnel.
- Le move est purement tactique et ne règle pas les problèmes structurels : la marque reste entachée par les controverses (CSAM, MechaHitler, surveillance), et xAI manque de crédibilité face à la concurrence (Anthropic, OpenAI).
- Les problèmes de confiance persistent : les utilisateurs rapportent une réduction drastique des quotas après paiement, une mauvaise transparence tarifaire, et des craintes que Grok Code soit tout aussi restrictif.
