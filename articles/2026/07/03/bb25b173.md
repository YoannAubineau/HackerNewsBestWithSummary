## Résumé de l'article

crustc est un compilateur Rust fonctionnel généré en code C, compilable avec GCC et make. Il s'agit d'une démonstration du projet cilly, une chaîne d'outils qui traduit le code Rust en C pour cibler des plateformes alternatives.

- cilly est un backend de compilateur Rust qui génère du code C adapté à chaque compilateur C cible, en interrogeant les capacités et les formats spécifiques du compilateur (tailles de types, encodages, support des mots-clés).
- Le projet vise à apporter le support Rust aux systèmes matériels anciens ou obscurs dépourvus de support LLVM ou GCC, tout en maintenant une compatibilité ABI avec le code compilé par rustc classique.
- cilly fonctionne de manière transparente réseau et permet de compiler pour des cibles cross-platform en utilisant n'importe quel compilateur C, y compris sur des systèmes exotiques comme Plan 9.
- La démonstration crustc compile le compilateur Rust lui-même en C (avec des wrappers C++ pour LLVM), généré pour ARM64 Linux avec GCC 13.3.0, et se construit en quelques minutes sans optimisations.
- Le projet n'est pas encore prêt pour une utilisation publique ; l'auteur prévoit une version complète de cilly une fois les problèmes liés aux optimisations résolus.

## Discussion sur Hacker News (66 commentaires)

**Avis positifs** :
- Projet impressionnant montrant une détermination remarquable à explorer un cas d'usage niche pendant 3 ans, avec respect de la dédicationportalité multi-plateforme: transpiler vers C permet de cibler tous les systèmes ayant un compilateur C, sans effort répété pour chaque architecture comme avec LLVM/GCC
- Élargit l'accès au débogage des portabilité: rend les problèmes accessibles aux ingénieurs C expérimentés (compilateurs, ABI, chaînes d'outils), pas seulement aux experts Rust/LLVM, facilitant ainsi la maintenance et le portage
- Cas d'usage légitime pour systèmes anciens/obscurs (Plan 9, matériel 32-bit) sans support LLVM/GCC moderne, ou avec documentation insuffisante pour ajouter du support à LLVM
- Permet le bootstrapping de Rust sur nouvelles plateformes et résout le problème circulaire (besoin d'un compilateur Rust pour compiler Rust), complétant les efforts existants comme mrustc
- Transpilation vers C (plutôt que LLVM IR) techniquement plus simple et pragmatique: C est une cible plus stable et universelle que d'autres intermédiaires

**Avis négatifs** :
- Peu utile pour très vieux ordinateurs (32-bit, rétro): Rust en tant que compilateur consomme trop de mémoire, limitation fondamentale non résolue par cette transpilation
- Porte seulement le compilateur rustc vers la plateforme cible, pas les programmes Rust; vous devez encore enseigner à LLVM comment cibler cette plateforme pour compiler du code Rust destiné à cette architecture
- Ne résout pas le problème de confiance 'Trusting Trust' si cilly est compilé avec rustc existant (potentiellement compromis); seul un mrustc indépendant peut l'éliminer
- Le code C généré s'apparente à un artefact/binaire plutôt qu'à du code source vérifiable, ne satisfait pas les standards du bootstrapping reproductible pour éliminer les attaques par backdoor
- L'ajout de support aux LLVM/GCC existants serait conceptuellement plus logique que de construire un traducteur complet, bien que techniquement plus ardu pour architectures exotiques mal documentées
