## Résumé de l'article

Copybara est un outil développé en interne par Google qui automatise la transformation et le déplacement de code source entre plusieurs dépôts Git. Il permet de maintenir synchronisés un dépôt confidentiel et un dépôt public, tout en gérant les contributions et les conflits de fusion.

- Fonctionne principalement avec Git (support expérimental de Mercurial) et nécessite un dépôt autoritaire unique comme source de vérité, tandis que les autres dépôts peuvent recevoir des contributions
- Stateless par design : stocke l'état dans le dépôt de destination via des labels dans les messages de commit, permettant à plusieurs utilisateurs d'obtenir des résultats identiques
- Cas d'usage courants : synchronisation régulière de code entre dépôts, import de sections confidentielles vers des dépôts publics, propagation de changements des dépôts non-autoritaires vers le dépôt autoritaire
- Disponible sous forme de binaires pré-compilés hebdomadaires ou compilable depuis les sources avec JDK 11 et Bazel
- Supporte les déploiements Docker avec variables d'environnement pour configurer le flux de travail (subcommande, fichier de config, workflow)

## Discussion sur Hacker News (46 commentaires)

**Avis positifs** :
- Outil très utile pour exporter des parties de monorepo vers GitHub et maintenir la synchronisation, particulièrement pour les projets open source internes.
- Effectue des transformations de code sophistiquées (remappage de chemins, exclusion de fichiers, nettoyage d'en-têtes) que git subtree ne peut pas faire, notamment pour adapter la structure et les systèmes de build entre repos.
- Scaling bien supérieur à git subtree sur les gros dépôts ; la communauté Rust a validé cette approche en créant Josh pour remédier aux problèmes de performance.
- Cas d'usage solide pour syncer une branche upstream externe ou une fork privée dans un monorepo interne, bien au-delà du simple export OSS.
- Améliore significativement la productivité des équipes collaborant dans un monorepo avec des portions publiques, avec un retour d'expérience positif d'utilisateurs Google.

**Avis négatifs** :
- Performances inacceptables en pratique pour certains cas d'usage ; des scripts bash avec git-filter-repo se sont avérés plus efficaces, notamment pour l'export de nombreux commits.
- La synchro bidirectionnelle devient très complexe quand les deux repos divergent : les transformations ne s'inversent pas proprement et Copybara génère des SHA différents, créant de la confusion.
- Outil complexe et potentiellement excessif pour un simple miroir entre repos sans transformations ; un simple mirroring GitLab/GitHub suffirait.
- Risque de devenir orphelin comme d'autres produits Google (Kaniko) ; moins critique que prévu si on cherche juste une synchro passive.
- Ne constitue qu'une solution de contournement au vrai problème : extraire et maintenant des bibliothèques réutilisables est la bonne pratique, copier du code est pragmatique mais pas idéal.
