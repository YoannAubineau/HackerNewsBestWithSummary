## Résumé de l'article

Terry Tao, mathématicien renommé, a utilisé des agents de codage basés sur les modèles de langage (LLM) pour moderniser ses outils numériques de visualisation mathématique. Les applets Java qu'il avait créés dès 1999 pour visualiser des objets mathématiques complexes (comme les nids d'abeille ou les ensembles de Besicovitch) ne fonctionnaient plus sur les navigateurs modernes ; en quelques heures, un agent IA a réussi à les porter en JavaScript avec quelques améliorations graphiques.

- Environ deux douzaines d'anciennes applets ont été migrées avec succès, révélant seulement un bug mineur et deux bugs dans le code original que l'agent a identifiés.
- Au-delà du portage, Tao a collaboré avec l'agent IA pour créer de nouveaux outils : une applet de visualisation pour la relativité restreinte (projet inachevé depuis 1999) et une applet pour la conjecture de Gilbreath accompagnant un récent article.
- Le processus s'est avéré suffisamment pratique pour que Tao envisage d'ajouter régulièrement de telles visualisations interactives aux futurs articles, considérant que le risque de bugs est acceptable pour des outils secondaires qui ne sont pas critiques pour le contenu mathématique principal.

## Discussion sur Hacker News (65 commentaires)

**Avis positifs** :
- Les agents LLM constituent un outil pratique et efficace pour générer du code auxiliaire non critique (visualisations, dashboards, interfaces pédagogiques), libérant du temps pour les tâches importantes.
- Il existe une demande latente infinie pour des logiciels, notamment en dehors des domaines traditionnels du software ; les LLM pourraient débloquer des années de travail en retard.
- L'utilisation de LLM pour les visualisations mathématiques et pédagogiques améliore significativement l'apprentissage et rend le contenu ancien plus accessible aux étudiants modernes.
- Les agents réduisent l'activation énergétique pour commencer et terminer des projets, permettant même aux experts de se concentrer sur leur domaine principal plutôt que sur la programmation.
- Même pour la recherche mathématique sérieuse, Tao a montré que les LLM peuvent assister à des tâches de preuve formelle et de découverte mathématique, pas seulement des projets hobbies.

**Avis négatifs** :
- Confier du code critique à des LLM comporte des risques : le code généré est souvent peu maintenable, peu fiable, et dépourvu de garanties formelles, contrairement à ce que certains affirment.
- Les commentaires rappellent que dire « ne pas faire confiance » aux LLM est imprecis : il faudrait plutôt reconnaître qu'on ignore encore distinguer les cas d'usage viables de ceux dangereux.
- Des craintes concernant les risques d'impact disruptif sur l'emploi des programmeurs et la qualité des salaires, particulièrement pour les jeunes générations, dominent la discussion.
- Certains questionnent l'absence de conflit d'intérêt apparent de Tao et notent que les articles de domaines experts utilisant l'IA restent des projets secondaires, jamais véritablement critiques.
- Les LLM ne sont que des « token vomitors » statistiques sans véritable autonomie ou avancée conceptuelle ; anthropomorphiser leur capacité via des termes comme « confiance » masque leur nature stochastique fondamentale.
