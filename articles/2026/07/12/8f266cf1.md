## Résumé de l'article

Une comparaison détaillée de l'utilisation de tokens entre Claude Code et OpenCode, deux harnesses d'agents de code basés sur le même modèle, révèle que Claude Code consomme environ 4,7 fois plus de tokens sur Sonnet 4.5 et 3,3 fois plus sur Claude Fable 5 avant même que l'utilisateur n'écrive sa première instruction. La mesure a été effectuée en intercalant un proxy de journalisation entre les harnesses et l'endpoint du modèle pour capturer exactement ce qui était envoyé.

- Claude Code envoie ~33 000 tokens de surcharge système (schémas d'outils, rappels, scaffolding) sur sa première requête contre ~7 000 pour OpenCode, l'écart se réduisant à 3,3x sur Claude Fable 5 en raison de prompts système model-dépendants
- OpenCode maintient des préfixes identiques byte-par-byte à chaque requête, ce qui permet une mise en cache efficace, tandis que Claude Code réécrit des dizaines de milliers de tokens de cache en session, générant jusqu'à 54x plus de tokens de cache à tarif premium
- Les fichiers de configuration réels (instruction files de 72KB, serveurs MCP) ajoutent respectivement ~20 000 et 5 000-7 000 tokens supplémentaires par requête, portant les sessions réelles à 75 000-85 000 tokens avant toute entrée utilisateur
- Le fan-out vers des sous-agents multiplie les coûts par 4,2x en raison du bootstrapping répété et de l'ingestion des transcriptions ; Claude Code a atteint 513 000 tokens pour une tâche qui en coûtait 121 000 directement
- Sur une tâche multi-étapes séquentielle, Claude Code a compensé sa surcharge initiale en regroupant les appels d'outils en moins de requêtes, convergeant vers les totaux d'OpenCode malgré un baseline plus élevé

## Discussion sur Hacker News (170 commentaires)

**Avis positifs** :
- Le système de cache d'Anthropic rend le coût initial des 33k tokens moins impactant après la première requête, puisque les tokens en cache sont facturés 10x moins cher.
- Claude Code offre une meilleure orchestration pour les tâches parallèles complexes et les sous-agents, malgré une consommation de tokens plus élevée.
- Les nouveaux modèles (Fable, Opus 4.8) ont des performances supérieures qui peuvent justifier la consommation additionnelle de tokens pour certaines tâches complexes.
- La verbosité accrue des nouveaux modèles peut refléter une meilleure qualité et une plus grande complétude des réponses plutôt qu'un simple gaspillage.
- OpenCode et Pi offrent des alternatives viables et moins coûteuses, prouvant que l'écosystème des harnesses se diversifie efficacement.

**Avis négatifs** :
- Claude Code envoie 33k tokens avant de lire la première requête (vs 7k pour OpenCode), créant un surcoût injustifié et opaque pour l'utilisateur.
- Anthropic contrôle le harness et les modèles, créant un conflit d'intérêts structural pour minimiser les coûts tokens des utilisateurs en abonnement.
- Les sous-agents de Claude Code consomment chacun l'intégralité du système prompt (33k tokens) sans partage de cache entre agents, rendant les tâches parallèles prohibitives.
- L'augmentation progressive de la consommation de tokens entre versions (4.6 → 4.8) suggère un pattern d'inflation tokens délibéré plutôt qu'une amélioration naturelle.
- Anthropic refuse l'utilisation des abonnements avec d'autres harness, forçant les utilisateurs à payer pour Claude Code même s'ils préfèrent OpenCode ou Pi.
