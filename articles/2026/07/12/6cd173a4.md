## Résumé de l'article

Mindwalk est un outil open-source qui visualise et rejoueur les sessions d'agents de codage (Claude Code, Codex) sur une carte 3D du dépôt. L'outil affiche où l'agent a exploré, lu et modifié du code, permettant de comprendre sa compréhension de la tâche en un coup d'œil.

- Fonctionne localement via un binaire Go unique ; les données de session ne quittent pas la machine
- Propose plusieurs vues (arborescence radiale, treemap) avec indicateurs visuels : fichiers non visités (sombre), consultés (vert mousse), lus (blanc lune), modifiés (ambre chaud)
- Lecture interactive avec scrubbing temporel, navigation par clics, raccourcis clavier et suivi de l'historique des visites par fichier
- Architecture modulaire en Go (trace, citymap, serveur) avec frontend React/Three.js, codifiée avec schémas JSON exportés
- Installation simple via script ou depuis GitHub Releases ; développement possible avec `make setup` et `make serve`

## Discussion sur Hacker News (62 commentaires)

**Avis positifs** :
- L'interface spatiale 3D représente une innovation prometteuse pour l'interaction avec les agents IA, rappelant les avancées conceptuelles de Xerox PARC et offrant une approche novatrice pour inspecter les capacités des LLM.
- Le cas d'usage principal de comparaison des performances entre différents modèles LLM sur une même tâche ou moyennes sur 100 exécutions est pertinent pour évaluer la variance et la cohérence des agents.
- L'outil offre un moyen plus rapide de visualiser les modifications distribuées dans une grande base de code comparé à parcourir du texte, permettant d'identifier immédiatement les zones affectées (rendu, base de données, serveur, etc.)
- La communauté apprécie l'aspect esthétique et créatif du projet, voyant une valeur dans la visualisation novatrice même si les cas d'usage pratiques restent à démontrer.
- Des possibilités de complémentarité émergent avec d'autres outils de visualisation 3D et de traçage d'agents pour améliorer le rendu et l'analyse.

**Avis négatifs** :
- La pertinence pratique reste contestée : une navigation 3D ajoute potentiellement plus de friction qu'elle n'en résout comparé aux outils standards (git diff, git log, IDE built-in), qui sont plus rapides et ergonomiques.
- Le projet manque de clarté sur le problème concret qu'il résout et comment la visualisation 3D fournit une solution actionnable au-delà de l'aspect esthétique.
- Les outils de visualisation de ce type tendent à être utilisés une seule fois ("c'est sympa, et ensuite?") puis oubliés, sans usage régulier ou justification d'existence à long terme.
- Des limitations pratiques existent : la vue en arborescence/terrain peut être arbitraire visuellement, la fonctionnalité d'historique est limitée par session plutôt que cumulative, et des données manquantes peuvent empêcher la visualisation.
- D'autres outils spécialisés (ctx.rs, gitai) répondent déjà mieux aux besoins réels comme le suivi de mémoire et de communication inter-agents, questionnant la valeur ajoutée de la replay.
