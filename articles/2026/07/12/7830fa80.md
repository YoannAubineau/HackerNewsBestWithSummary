## Résumé de l'article

Mesh LLM est un système distribué qui pool les GPU et la mémoire de plusieurs machines pour exécuter des modèles de langage, en les exposant via une API compatible OpenAI. Au lieu de dépendre d'un fournisseur centralisé, les utilisateurs conservent le contrôle sur leurs données, leur matériel et leurs coûts, tout en bénéficiant d'une infrastructure pair-à-pair sécurisée construite sur iroh.

- Le système permet de diviser des modèles trop volumineux pour une seule machine (mode "Skippy") en les partitionnant par couches sur plusieurs nœuds, dont les activations circulent en pipeline
- Chaque nœud démarre un point de terminaison iroh qui gère l'authentification, le traversal NAT et les connexions QUIC directes, sans serveur central
- Les requêtes peuvent être traitées localement, routées vers un pair ayant le modèle chargé, ou distribuées dans le pipeline, de manière transparente pour le client
- Le logiciel léger (18 MB) offre un catalogue de 40+ modèles (de 500M à 235B paramètres) et s'intègre avec des clients OpenAI standard via localhost:9337/v1
- Une application mobile basée sur le SDK Swift d'iroh est en développement, avec support prévu pour ACP, l'émergent standard agent pour élargir l'écosystème pair-à-pair

## Discussion sur Hacker News (50 commentaires)

**Avis positifs** :
- Permet le calcul distribué sans matériel personnalisé dédié, en réutilisant des machines existantes (serveurs, stations de travail, etc.)
- Les performances peuvent être compétitives : 16 tokens/s observés sur des modèles large en conditions réelles, approchant l'interactivité acceptable
- Architecture intelligente : les poids restent en VRAM sur chaque machine, seules les activations (kilobytes) transitent par le réseau, réduisant le bottleneck réseau comparé au swapping disque/RAM
- Bénéfices majeurs en phase de préfill (traitement du prompt) où la latence réseau n'est pas cumulative, et potentiel avec spéculative decoding
- Chiffrement intégré via iroh (QUIC avec clés d'endpoints), intérêt écosystémique croissant avec d'autres projets similaires

**Avis négatifs** :
- Performances limitées pour usage interactif : la latence réseau se cumule à chaque saut entre nœuds, créant un plafond théorique (ex : 30 tokens/s avec 1ms de latence sur 4 nœuds)
- Impraticable sur internet public en raison de latences trop élevées ; limité aux réseaux locaux/d'entreprise avec faible latence
- Problèmes de sécurité et confidentialité non résolus : tous les acteurs voient les séquences traitées, risque d'empoisonnement par des acteurs malveillants, ni protection spécifique contre les attaques RCE
- Manque de documentation et de benchmarks complets : absence d'information sur les conditions de test, matériel exact, quantification, rendant difficiles les comparaisons réelles
- Implémentation immature avec bugs d'installation, limitation actuelle du support AMD, absence de garanties d'équité pour les contributeurs de ressources (pas de modèle d'incitation/compensation)
