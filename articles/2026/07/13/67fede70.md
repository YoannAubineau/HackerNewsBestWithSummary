## Discussion sur Hacker News (102 commentaires)

**Avis positifs** :
- Grok CLI a téléchargé le répertoire utilisateur en entier de façon préemptive et non-consentie, ce qui est un dysfonctionnement grave indépendamment du débat sur les sandboxes
- Les utilisateurs non-techniques ne peuvent pas raisonnablement s'attendre à ce qu'un outil télécharge automatiquement leurs données sensibles ; c'est de la responsabilité du fabricant d'avoir des comportements sûrs par défaut
- D'autres outils (Claude, Copilot) ont des antécédents similaires de vol de données ; c'est un problème systémique chez les fournisseurs IA fermés qui priorisent l'absorption de données
- Les entreprises big tech utilisent systématiquement ces agents pour exfiltrer du code propriétaire et des clés secrètes ; une approche opt-in aurait été appropriée pour une fonctionnalité aussi intrusive

**Avis négatifs** :
- L'utilisateur a explicitement configuré son répertoire home (~/) comme repo_path de confiance, ce qui rend le comportement logique plutôt qu'une trahison cachée
- Les modèles LLM doivent bien envoyer le contexte vers les serveurs du fournisseur pour fonctionner ; il n'y a pas de magie et pas d'alternative téchniquement viable pour les agents en cloud
- Les comparaisons avec Dropbox ou les navigateurs de fichiers locaux sont trompeuses car Grok CLI fonctionne sur les serveurs du fournisseur, pas localement
- Des alternatives existent (sandboxing, conteneurs, modèles locaux, outils open-source) ; le blâmer exclusivement le fournisseur sans responsabilité utilisateur minimalise les options disponibles
- La majorité des utilisateurs d'IA ne suivent pas HN ou X et n'ont pas connaissance de ces incidents récurrents ; attendre une vigilance universelle face à des problèmes moins médiatisés est irréaliste
