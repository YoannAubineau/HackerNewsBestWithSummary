## Résumé de l'article

DeepClaude est un outil qui remplace le modèle API utilisé par Claude Code (agent autonome de programmation d'Anthropic) en utilisant DeepSeek V4 Pro via un proxy local, réduisant les coûts de 17 fois tout en conservant la même interface et fonctionnalités.

- Utilise DeepSeek V4 Pro à $0.87/million tokens au lieu d'Anthropic à $15/million, avec mise en cache automatique réduisant le coût à $0.004/million pour les requêtes suivantes
- Fonctionne via un proxy local (localhost:3200) qui intercepte les appels API et permet de basculer entre DeepSeek, OpenRouter, Fireworks AI et Anthropic sans redémarrage
- Conserve toutes les fonctionnalités de Claude Code : édition de fichiers, exécution bash, boucles autonomes multi-étapes, subagents et opérations Git
- Permet le basculement de backend en cours de session depuis la ligne de commande ou directement depuis Claude Code avec des slash commands
- Compatible avec le mode distant (remote control) en passant par le pont WebSocket d'Anthropic tout en acheminant les appels modèle vers DeepSeek, nécessitant une authentification et une souscription Claude.ai

## Discussion sur Hacker News (111 commentaires)

**Avis positifs** :
- La configuration est triviale : il suffit de définir quelques variables d'environnement pour utiliser DeepSeek avec Claude Code, ce qui rend le projet immédiatement accessible
- DeepSeek V4 Pro offre un excellent rapport qualité-prix comparé à Claude Opus, avec plusieurs utilisateurs confirmant des résultats satisfaisants pour des tâches de codage
- L'approche de combiner un modèle premium pour la conception (Opus/GLM-5.1) et un modèle moins cher pour l'implémentation (DeepSeek/Qwen) permet des économies substantielles sans sacrifier la qualité
- Le proxy local pour basculer entre modèles en cours de session et le suivi des coûts combinés ajoutent de véritables fonctionnalités utiles au-delà de la simple substitution
- D'autres outils comme Hermes, Langcli et OpenCode offrent maintenant des alternatives compétitives avec support natif de DeepSeek et meilleure compatibilité avec les API ouvertes

**Avis négatifs** :
- Le projet consiste essentiellement en quelques lignes de variables d'environnement—une documentation Anthropic et DeepSeek officielle rend ce 'wrapper' largement inutile
- Le readme a probablement été généré par une IA et ne met pas en avant les véritables fonctionnalités, tandis que le commit git révèle un plan marketing agressif suspecte
- Sonnet et les modèles équivalents de DeepSeek échouent encore sur des tâches complexes et nécessitent une intervention manuelle d'Opus, annulant les économies prétendues
- DeepSeek n'offre pas la possibilité d'opt-out pour l'entraînement, et les préoccupations concernant la confidentialité des données soulèvent des doutes sur les projets critiques
- Le temps passé à jongler entre plusieurs outils et configurations rivaux dépasse les gains économiques—rester avec Claude Code évite la fatigue de choisir et de rester à jour constamment
