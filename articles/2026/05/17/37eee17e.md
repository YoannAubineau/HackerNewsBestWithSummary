## Résumé de l'article

Un développeur a créé un serveur web fonctionnant sur un microcontrôleur AVR64DD32, un petit processeur 8 bits de 24 MHz avec 32 KB de mémoire. Connecté à internet via SLIP (Serial Line Internet Protocol) sur un lien série USB, le serveur implémente les protocoles IP et TCP de manière minimaliste et héberge une page statique accessible publiquement à travers un mandataire VPN.

- Le microcontrôleur ne peut pas gérer l'Ethernet direct (trop rapide), donc l'auteur a utilisé SLIP, un ancien protocole réseau sur liaison série, toujours supporté par Linux moderne
- L'implémentation du protocole TCP a requis plusieurs jours de travail pour gérer les états de connexion, les retransmissions et les cas limites, tandis que HTTP a été simplifié en réponses codées en dur
- Pour accéder publiquement au serveur sans adresse IPv4 dédiée, l'auteur utilise WireGuard pour créer un tunnel vers un VPS en Finlande, puis configure un mandataire web pour router les requêtes sous `/mcu` vers le microcontrôleur
- Le serveur fonctionne avec une puissance minimale et peut s'alimenter directement du port série USB, nécessitant un seul câble de connexion
- Cette approche duplique le modèle du "Vape Server", un autre site hébergé sur un microcontrôleur récupéré à la casse

## Discussion sur Hacker News (20 commentaires)

**Avis positifs** :
- Le projet ravive la nostalgie des débuts d'Internet avec le chargement progressif du contenu, comme aux temps du dial-up
- Les microcontrôleurs modernes (AVR DD/EA/EB) offrent des périphériques impressionnants (ADC 12-bit avec gain programmable, MVIO, système d'événements) qui les rendent viables pour des applications web
- C'est une continuation d'une tradition de 25+ ans de minimalisme : des projets historiques comme le serveur web sur puce ACE1101 (moins de 1024 bytes) montrent que c'est faisable et fascinant
- La solution est pratique en production : le client gère le rendu lourd tandis que le contrôleur livre simplement le contenu depuis une carte SD

**Avis négatifs** :
- L'AVR DD risque de devenir obsolète face au PIC32 CM ARM Cortex-M0+ pin-compatible, qui offre des performances et des périphériques comparables avec un cœur 32-bit standard
- La limitation de bande passante reste sévère : Ethernet 10BASE-T ne peut théoriquement atteindre que ~10 Mbps utiles après encodage Manchester, ce qui est très restrictif
- Microchip semble pivotter progressivement vers ARM pour les futures générations plutôt que de continuer à développer des plates-formes 8-bit, suggérant l'obsolescence du marché 8-bit
