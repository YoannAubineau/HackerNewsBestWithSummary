## Résumé de l'article

rlisp est un projet de fin de semaine qui permet d'écrire du code Rust en utilisant la syntaxe s-expression de Lisp. Le compilateur transpile les s-expressions en code source Rust (.rs) que rustc compile ensuite, conservant tous les mécanismes de Rust (vérification de types, borrow checking, génériques, traits) mais avec une syntaxe uniforme basée sur les listes imbriquées.

- Les macros Lisp standard (defmacro, quasiquote, unquote) offrent une alternative aux proc_macro de Rust sans dépendre de syn/quote
- Toutes les fonctionnalités Rust sont supportées : ownership, borrowing, lifetimes, pattern matching, unsafe blocks, modules, et visibilité (pub, pub(crate), etc.)
- La syntaxe uniforme élimine les délimiteurs inconsistants des braces et permet l'édition structurelle ; les identifiants en kebab-case se convertissent automatiquement avec des underscores doubles
- Un escape valve (rust "...") permet d'injecter du code Rust brut quand la syntaxe Lisp ne suffit pas
- L'outil offre trois commandes : rlisp compile (transpile uniquement), rlisp build (transpile + rustc), rlisp run (exécution complète)

## Discussion sur Hacker News (66 commentaires)

**Avis positifs** :
- Permet d'exprimer la sémantique de Rust (ownership, lifetimes, memory safety) via une syntaxe s-expression, ce qui intéresse ceux qui apprécient les garanties de Rust mais trouvent sa syntaxe verbeux ou complexe.
- Offre un point d'accès intéressant pour les macros Lisp (transformations au niveau AST) tout en conservant la sécurité mémoire et la performance de Rust, contrairement aux Lisps classiques.
- Démontre que des outils comme Claude/DeepSeek peuvent générer efficacement du code s-expression bien-formé avec parenthèses appairées, contredisant les inquiétudes sur les LLMs et cette syntaxe.
- S'inscrit dans une lignée de projets légitimes explorant Lisp + ownership (Carp, Loon), montrant l'intérêt croissant pour combiner sécurité mémoire et expressivité du Lisp.
- Constitue un point de départ utile pour accélérer les branchements sur Rust-comme-syntaxe-alternative, même si encore incomplet (gestion des lifetimes HRTB, turbofish, messages d'erreur détaillés).

**Avis négatifs** :
- Le projet est incomplet et contenait des bugs de compilation (span inutiles, Display non implémentés) ; assume incorrectement que les types monolettres sont toujours des paramètres génériques.
- Qualifié d'« LLM slop » par plusieurs commentateurs ; la création assistée par IA soulève des questions sur la qualité intrinsèque et le manque d'intérêt pour un sujet aussi technique sans réelle innovation.
- N'est qu'un wrapper syntaxique sur Rust existant, pas une véritable langue Lisp ; GalaxyNova et d'autres notent que c'est « weird-looking » même pour les programmeurs Lisp, sans sémantique innovante.
- Manque des syntaxes Rust complexes critiques (lifetimes complètes, turbofish, HRTB) et ne dispose pas de LSP/rust-analyzer pour IDE, rendant impropre à un usage réel ; ne montre pas d'exemples concrets de ces cas.
- La justification « no runtime, no GC, just » est un marketing maladroit et techniquement confus (pourquoi un parseur syntaxique introduirait un GC ?) ; README peu convaincant pour justifier l'apparition en première page.
