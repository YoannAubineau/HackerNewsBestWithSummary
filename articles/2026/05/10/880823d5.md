## Résumé de l'article

GitHub, plateforme de gestion de code source détenue par Microsoft depuis son acquisition, fait face à des critiques croissantes concernant sa fiabilité, ses performances et sa direction. Plusieurs projets notables annoncent leur départ de la plateforme, tandis que l'auteur souligne que Git lui-même reste un outil open-source indépendant de GitHub.

- L'article dénonce une dégradation générale de GitHub sous Microsoft, avec des problèmes d'uptime, une prolifération de contenu automatisé (« slop ») et des services comme GitHub Copilot jugés problématiques
- Git et GitHub ne sont pas synonymes : Git est une technologie distribuée open-source qui fonctionne sans service centralisé, tandis que GitHub n'est qu'une plateforme au-dessus
- Plusieurs alternatives centralisées sont présentées : Codeberg (non-profit), GitLab, Gitea, Tangled, ou l'auto-hébergement avec Forgejo
- L'auto-hébergement d'une forge Git ou l'utilisation de Git brut (via SSH) restent des options viables et ne nécessitent pas de migration complète en une seule opération
- L'auteur recommande de commencer progressivement à migrer vers d'autres solutions plutôt que d'attendre l'effondrement complet de la plateforme

## Discussion sur Hacker News (120 commentaires)

**Avis positifs** :
- Les limites de taux non authentifiées de GitHub fonctionnent comme un blocage par défaut, pas une vraie limitation technique, et le message d'erreur induit en erreur les utilisateurs sur la nature du problème
- L'augmentation 10x du code committé via l'IA (vibe-coding, agents) a créé une charge d'infrastructure sans précédent que GitHub n'a pas correctement dimensionnée ou monétisée
- Les données de disponibilité historique de GitHub sont inexactes et sous-déclarées, masquant la véritable gravité des pannes passées
- Microsoft a dégradé GitHub depuis son acquisition (2018) : infrastructure Azure performante, cerveaux d'ingénieurs partis, intégrations Copilot mal gérées
- Le modèle économique freemium de GitHub ne peut pas supporter 100x de trafic sans augmenter les revenus, ce qui pourrait faire fuir les utilisateurs payants

**Avis négatifs** :
- Les pannes de GitHub ont commencé en 2020 avec une dégradation progressive jusqu'en 2023, bien avant l'inflation d'usage IA massif (fin 2024-2025), contredisant la théorie de l'IA comme cause unique
- GitHub Actions est une amélioration positive depuis l'acquisition, tout comme l'élargissement du plan gratuit aux dépôts privés illimités et les outils de gestion de projet améliorés
- GitLab, Bitbucket et autres alternatives n'ont pas connu les mêmes pics de défaillance malgré l'existance d'outils IA génériques, suggérant que le problème n'est pas l'IA généralisée mais spécifique à GitHub
- Microsoft dispose de ressources cloud massives (Azure, 200k employés) pour maintenir GitHub : blâmer l'IA plutôt que l'incompétence managériale ou les choix d'architecture revient à chercher des excuses
- Les utilisateurs avec des alternatives viables (self-hosted Forgejo, Gitea, Codeberg) montrent qu'une décentralisation fonctionnelle est possible et préférable sans nécessiter GitHub
