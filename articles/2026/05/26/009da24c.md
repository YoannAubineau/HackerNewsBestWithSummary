## Résumé de l'article

Un article d'opinion d'un développeur qui estime que React est souvent mal utilisé, malgré ses cas d'usage légitimes.

- L'auteur considère que React est « presque toujours la mauvaise solution » pour la plupart des projets
- Il reconnaît que React a sa place mais est devenu un outil appliqué à tous les problèmes sans discernement
- Selon lui, bien que React puisse être mis en œuvre correctement, cela arrive rarement en pratique

## Discussion sur Hacker News (311 commentaires)

**Avis positifs** :
- React offre une abstraction mentale claire et cohérente : composants comme fonctions d'état, absence de magie framework, et JSX comme DSL modulaire et expressive qui fonctionne indépendamment de React.
- React excelle pour les applications interactives complexes et les rich UIs où le modèle déclaratif et basé sur les composants surpasse la manipulation directe du DOM, particulièrement avec des bibliothèques modernes comme Tanstack Query.
- L'écosystème React est vastement supérieur en taille et maturité, offrant des bibliothèques, des outils et des ressources que les alternatives ne peuvent égaler, ce qui représente un avantage professionnel et économique réel.
- React a marqué une rupture positive dans l'évolution du développement frontend en introduisant un paradigme qui a influencé toutes les frameworks modernes (Vue, Svelte, Solid) et apporté une clarté conceptuelle sur le rendu d'UI.
- Pour les développeurs venant de jQuery ou AngularJS 1, React a résolu des problèmes fondamentaux de gestion d'état et de rendu, offrant une expérience de développement plus intuitive et maintenable.

**Avis négatifs** :
- React force les développeurs dans l'écosystème JavaScript/TypeScript et crée un coût de complexité énorme : virtualDOM, hooks (useState, useEffect), compilateurs, et chaînes de build complexes pour résoudre des problèmes que des alternatives plus simples n'ont pas.
- Les hooks (surtout useEffect) sont une abstraction qui fuit et source d'erreurs subtiles ; le modèle de réactivité implicite crée des bugs difficiles à déboguer comparé aux alternatives comme Vue (signaux) ou les composants de classe traditionnels.
- React encourage une culture d'over-engineering : la majorité des sites web n'ont pas besoin d'une SPA ou de React, simples serveurs avec HTML rendu côté serveur + JavaScript ciblé suffiraient avec meilleure performance et accessibilité.
- La domination de React repose sur l'inertie (tous les autres l'utilisent donc j'apprends React) plutôt que sur les mérites techniques ; alternatives comme Svelte, Solid.js ou Alpine.js offrent une meilleure DX et performance pour beaucoup d'usages.
- Depuis l'acquisition par Vercel, React et Next.js ont dévié vers des «solutions magiques» (Server Components, etc.) qui réintroduisent de la complexité opaque et diminuent la flexibilité pour les développeurs.
