## Résumé de l'article

Cette liste curatée recense les principaux ouvrages sur la programmation CUDA, technologie de programmation parallèle sur processeurs graphiques NVIDIA. Elle couvre tous les niveaux d'expertise et inclut les publications de 2024-2026.

- **Classement par thème** : débutants, architecture GPU, guides pratiques, optimisation avancée, Python, et titres récents (2022-2026)
- **Classiques incontournables** : « CUDA by Example » (2010) pour débuter, « Programming Massively Parallel Processors » (3e édition, 2022) comme référence d'architecture, « The CUDA Handbook » (2013) pour les détails bas niveau
- **Ressources modernes** : « Programming in Parallel with CUDA » (Ansorge, 2022) axé C++ et applications scientifiques réelles, « CUDA for Deep Learning » (Arledge, 2025) couvrant l'optimisation pour le deep learning
- **Pour utilisateurs Python** : « Hands-On GPU Programming with Python and CUDA » (Tuomanen, 2018) avec Numba, CuPy et bindings
- **Contributions bienvenues** : le projet accepte les propositions de nouveaux livres post-2018 ou classiques toujours pertinents via pull requests sur GitHub

## Discussion sur Hacker News (52 commentaires)

**Avis positifs** :
- Les ressources officielles NVIDIA et vidéos d'architectes CUDA (comme Stephen Jones) offrent une compréhension solide des concepts fondamentaux qui n'ont pas drastiquement changé
- L'apprentissage par la pratique sur du code réel (kernels Flash Attention, vLLM) est plus efficace que les livres abstraits pour comprendre la hiérarchie mémoire
- Les connaissances de base (device/host boundary, SIMT programming) restent stables et fiables depuis ~10 ans, un bon investissement même avec des livres de 2012
- Des outils modernes comme NVIDIA Warp permettent d'écrire des kernels CUDA directement en Python, réduisant la barrière d'entrée
- Utiliser des bibliothèques de haut niveau plutôt que d'écrire ses propres kernels est souvent la bonne décision pour la plupart des cas d'usage

**Avis négatifs** :
- La plupart des livres existants (même PMPP) contiennent des erreurs de code, une écriture confuse et ne reflètent pas les paradigmes modernes (cuTile, tensor cores, async DMA)
- Les kernels performants pour le matériel NVIDIA moderne ressemblent peu à ce que les livres de 2012 enseignent—les connaissances avancées deviennent rapidement obsolètes
- La fragmentation des outils NVIDIA (Warp, Numba CUDA, CUDA Python) crée de la confusion pour les débutants sur le meilleur chemin d'apprentissage
- Lire des livres sur CUDA en contexte professionnel est difficile quand l'entreprise valorise la productivité immédiate avec les LLMs plutôt que l'apprentissage profond
- NVIDIA ne prioritise pas le support complet pour tous les GPU (ex: Blackwell sans kernels complets pour SM 120), ce qui encourage l'enfermement propriétaire
