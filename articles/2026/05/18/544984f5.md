## Résumé de l'article

CUDA Books est un répertoire organisé de ressources littéraires majeures couvrant la programmation parallèle sur GPU NVIDIA, du niveau débutant aux techniques avancées d'optimisation. La liste inclut des ouvrages classiques, des guides pratiques en C++ et Python, ainsi que les publications les plus récentes de 2024-2026.

- **Ouvrages fondamentaux** : « CUDA by Example » (Sanders & Kandrot, 2010) reste la référence d'introduction ; « Programming Massively Parallel Processors » (Kirk & Hwu, 3e édition 2022) est le standard universitaire d'architecture GPU.
- **Catégories couverte** : débutants, architecture et programmation parallèle, guides pratiques, optimisation avancée, développement Python (Numba, CuPy), et publications récentes spécialisées (deep learning, debugging, C++20).
- **Ressources pratiques modernes** : « CUDA for Deep Learning » (Arledge, 2025) enseigne l'optimisation pour réseaux de neurones ; « GPU Programming with C++ and CUDA » (Motta, 2024) couvre C++20 et l'interopérabilité Python.
- **Recommandation complémentaire** : la liste suggère de coupler les livres avec le guide officiel gratuit CUDA C++ Programming Guide v13.x pour rester à jour face aux évolutions rapides de la technologie.
- **Contributions communautaires** : le projet accepte les propositions pour ajouter des titres de qualité parus après 2018 ou les classiques toujours pertinents, via pull requests sur le dépôt.

## Discussion sur Hacker News (52 commentaires)

**Avis positifs** :
- Les fondamentaux de CUDA (device/host boundary, programmation SIMT) restent stables depuis 10+ ans et valident les contenus pédagogiques classiques, même des livres de 2012
- L'apprentissage pratique via du vrai code (Flash Attention, vLLM) rend la hiérarchie mémoire concrète et plus efficace que les explications abstraites des livres
- Des ressources complémentaires modernes comme Warp (Python pour CUDA) ou les cours de l'ORNL facilitent l'accès et offrent des approches novatrices
- Un besoin réel d'amélioration : plusieurs lecteurs expérimentés demandent des ressources meilleures que les standards actuels (erreurs, écriture confuse)

**Avis négatifs** :
- Les livres anciens manquent les cores modernes (Tensor, raytracing) et les optimisations avancées (TMA, async memcpy DMA) essentielles pour la performance actuelle
- Écrire des kernels CUDA efficaces est très difficile ; la plupart des implémentations personnalisées performent mal—mieux vaut utiliser des bibliothèques haut-niveau
- Confusion croissante avec plusieurs outils Nvidia concurrents (Warp, Numba CUDA, CUDA Python) sans guidance claire pour les débutants
- Le verrouillage matériel Nvidia et l'incohérence (ex: SM120 sans kernels optimisés sur Blackwell) remettent en question la fiabilité de dépendre uniquement de Nvidia
- Le temps disponible pour lire est nul face aux pressions d'entreprise d'utiliser les LLM et les AI agents, réduisant l'intérêt pratique de ces ressources
