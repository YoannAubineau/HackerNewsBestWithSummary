## Discussion sur Hacker News (118 commentaires)

**Avis positifs** :
- Le jeu illustre efficacement la fatigue des permissions et comment elle pousse les utilisateurs à contourner les contrôles de sécurité, validant le problème identifié dans l'article
- Les commentaires confirment que les systèmes de permissions actuels sont mal conçus : soit les utilisateurs les contournent, soit ils approuvent tout par habitude, créant une fausse sécurité
- L'isolation en conteneurs/VMs est reconnue comme une approche pragmatique : les agents peuvent avoir moins de restrictions si l'environnement lui-même est sécurisé et réversible
- Le jeu met en lumière un problème réel : les modèles de menace sont souvent mal compris, et approuver des commandes individuelles est un mauvais primitif de sécurité
- Plusieurs utilisateurs confirment utiliser des alias pour ignorer les permissions dans leurs workflows réels, prouvant que la fatigue pousse à des comportements dangereux

**Avis négatifs** :
- Plusieurs critiques pointent que le jeu a une compréhension imparfaite des menaces réelles : il marque comme dangereux des commandes inoffensives (cat ~/.zshrc si les secrets ne s'y trouvent pas) et omet que le vrai danger vient de ce que l'agent peut écrire dans le projet
- Les commentaires soulignent que le jeu est trop JavaScript-centric et ne représente pas fidèlement les contextes réels : les menaces varient énormément selon la configuration et l'environnement de chacun
- Les permissions par commande ne sont pas le bon primitif : un agent peut écrire du code malveillant dans package.json ou les dépendances, et npm run build sera approuvé sans vérifier son contenu, rendant la permission inefficace
- Critiques du modèle de menace du jeu : il suppose que tout le monde stocke ses secrets en texte clair dans sa configuration shell, ce qui reflète de mauvaises pratiques plutôt qu'une réalité universelle
- L'approche de révision par un autre agent ("auto-mode") est présentée comme plus efficace et pratique que les permissions utilisateur, rendant le système du jeu obsolète par rapport aux solutions actuelles
