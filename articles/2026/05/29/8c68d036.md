## Résumé de l'article

Nightmare-Eclipse, un chercheur en sécurité, a vu son compte GitHub supprimé par Microsoft après avoir publié plusieurs exploits Windows zero-day sans passer par les canaux officiels. Eclipse dénonce une action punitive, affirmant que Microsoft a ignoré ses rapports de vulnérabilités et refusé de verser les primes de bug bounty du programme MSRC.

- Eclipse a développé au moins six exploits zero-day pour Windows (BlueHammer, RedSun, UnDefend, GreenPlasma, MiniPlasma, YellowKey), certains étant actuellement exploités en conditions réelles
- Le chercheur accuse Microsoft d'avoir refusé de communiquer et de ne pas honorer les paiements du programme de bug bounty (jusqu'à 250 000 dollars pour Hyper-V)
- Microsoft a également supprimé le compte Microsoft d'Eclipse utilisé pour signaler les failles, sans justification publique
- Eclipse a annoncé la publication de nouvelles exploits le 14 juillet, date coïncidant avec Patch Tuesday
- L'interdiction de GitHub est largement critiquée comme inefficace et contre-productive, puisque Eclipse a simplement migré vers GitLab et que les preuves de concept sont déjà publiques

## Discussion sur Hacker News (146 commentaires)

**Avis positifs** :
- Microsoft banning a security researcher crée un précédent dangereux qui dissuadera d'autres chercheurs de signaler les vulnérabilités de manière responsable, nuisant à la sécurité globale
- Les programmes de bug bounty de grandes entreprises comme Microsoft DOIVENT rémunérer équitablement les chercheurs ; refuser de payer alors qu'on accepte les exploits signalés détruit la confiance et légitime de vendre les 0-days ailleurs
- Microsoft possède GitHub, ce qui crée un conflit d'intérêt flagrant : l'entreprise peut cenurer du contenu qui l'embarrasse tout en restant une plateforme publique supposément neutre
- Le rejet du chercheur par Microsoft (refus de paiement, exigences bureaucratiques déraisonnables) justifie sa frustration ; se faire bannir après avoir signalé de sérieuses failles est une mauvaise réaction à un problème de sécurité réelle

**Avis négatifs** :
- Le chercheur a publié les exploits SANS notification préalable à Microsoft et a menacé l'entreprise (« broyer les os »), ce qui justifie un bannissement indépendamment d'un conflit de bounty
- Les exploits ont immédiatement été clonés et forkés partout ; bannir des comptes GitHub ne supprime rien, les théories de censure coordonnée entre Microsoft et GitLab (entreprises sans lien) semblent infondées
- Les allégations de complot gouvernemental ou de backdoor intentionnel manquent de preuves ; il est plus probable que l'exploit soit une limitation technique connue (TPM-only) plutôt qu'une conspiration
- Le chercheur a également été banni de GitLab (non lié à Microsoft) et affiche des comportements agressifs publiquement ; cela suggère un bannissement basé sur la conduite plutôt qu'une censure coordonnée
