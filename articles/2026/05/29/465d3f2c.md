## Résumé de l'article

Claude Code est un outil d'assistance au développement distribué comme paquet npm qui contient de nombreuses fonctionnalités avancées non documentées officiellement. Ces capacités cachées permettent de créer des systèmes d'autorisation personnalisés, des agents d'IA persistants dotés de mémoire, des scripts de gestion avancée, et des boucles d'auto-amélioration.

- Les hooks peuvent retourner du JSON structuré pour modifier les commandes en temps réel (updatedInput), forcer des décisions d'autorisation (permissionDecision), ou injecter du contexte additionnel, bien au-delà de ce que documente l'interface officielle
- Trois champs de hook non documentés (once, async, asyncRewake) permettent respectivement une exécution unique, un lancement non-bloquant, ou un blocage conditionnel avec réveil du modèle
- Le système de compétences (Skills) accepte des champs frontmatter cachés (model, effort, hooks, agent) pour contrôler le modèle utilisé, la profondeur de réflexion, et la délégation à des agents personnalisés
- Les agents personnalisés supportent la mémoire persistante par projet ou globalement (memory: project/user) et peuvent apprendre et consolider les connaissances via autoMemoryEnabled et autoDreamEnabled
- Le classificateur auto-mode accepte des descriptions en langage naturel de l'environnement (environment array) pour ajuster les décisions de sécurité plutôt que de simples patterns globaux

## Discussion sur Hacker News (58 commentaires)

**Avis positifs** :
- Claude Code offre une richesse de fonctionnalités de configuration qui méritent d'être mieux documentées et comprises par les utilisateurs
- L'article révèle des patterns architecturaux utiles (contexte, planification, exécution, vérification) convergents entre les agents IA de codage
- Les configurations non documentées comme showClearContextOnPlanAccept et disableAutoMode peuvent réellement améliorer les workflows existants
- La nécessité de guides détaillés sur Claude Code est réelle face à une documentation officielle fragmentée et difficile à naviguer

**Avis négatifs** :
- La majorité des fonctionnalités présentées comme « non documentées » figurent en fait dans la documentation officielle (hooks, skills frontmatter, auto-mode config)
- L'article date d'avril 2026 et est déjà partiellement obsolète (yoloClassifier remplacé par yoloEquivEnabled, « Magic Docs » disparus en v2.1.156)
- Compter sur des configurations non documentées est risqué : elles changent fréquemment (exemple : suppression de « clear context and execute plan » après Opus 1M)
- Les fonctionnalités mentionnées comme undocumented se révèlent souvent contrôlées par des feature flags ou documentées de manière fragmentée plutôt que véritablement cachées
- L'article semble généré par IA (confirmé par analyse Pangram), ce qui explique les inexactitudes et soulève la question de sa fiabilité
