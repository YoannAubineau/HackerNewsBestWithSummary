## Résumé de l'article

Red Squares est un projet qui représente les indisponibilités de GitHub sous la forme de carrés rouges sur le calendrier de contribution, parodiant ainsi le système de gamification des contributions de GitHub.

- Le projet transforme les statuts de panne de GitHub en marqueurs visuels sur le graphique de contribution habituel
- Il joue sur l'ironie d'afficher les défaillances du service comme s'il s'agissait d'activité productive
- L'idée a généré 379 points et 77 commentaires sur Hacker News, montrant l'intérêt pour cette satire technique

## Discussion sur Hacker News (83 commentaires)

**Avis positifs** :
- La corrélation entre les pannes et les jours de semaine suggère que les déploiements fréquents (toutes les 60-90 minutes) et les changements de code sont le principal facteur causant les instabilités, particulièrement mardi à jeudi
- Le volume exponentiel de commits (estimé à 14x entre quelques années) et l'introduction d'outils IA accélérant les déploiements automatisés créent une charge imprévisible et des instabilités proportionnelles
- L'absence quasi-totale de pannes le week-end renforce l'hypothèse que les problèmes proviennent de l'activité humaine (déploiements, changements) plutôt que de limitations matérielles pures
- La visualisation créative des pannes en contribution GitHub est astucieuse et ironique, révélant un vrai problème : GitHub affiche une fiabilité illégalement déclarée dans ses SLA comparée aux pages de statut indépendantes

**Avis négatifs** :
- GitHub n'a jamais eu une fiabilité optimale même avant 2020, et les données d'uptime à 100% avant le rachat Microsoft n'étaient dues qu'à une mesure imprécise, pas à une réelle dégradation récente
- Blâmer les agents IA et les outils de productivité est spéculatif ; les problèmes remontent à bien avant leur arrivée (2020), suggérant que l'infrastructure sous-jacente était déjà défaillante
- Comparer la fiabilité de GitHub public et GitHub Enterprise Cloud est un faux débat puisqu'ils partagent la même infrastructure ; la meilleure fiabilité observée sur Enterprise provient simplement de moins d'utilisateurs
- Une part significative des 'pannes' rapportées concernent uniquement les modèles IA (Gemini, Claude, Grok), dont GitHub n'est que le distributeur, ce qui gonfle artificiellement les statistiques sans refléter une défaillance GitHub elle-même
