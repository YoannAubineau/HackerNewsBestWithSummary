## Discussion sur Hacker News (147 commentaires)

**Avis positifs** :
- Les modifications générées par Claude concernent principalement la suite de tests et l'infrastructure CI, ce qui est un usage pertinent de l'IA pour identifier des lacunes et renforcer la couverture de test.
- Les correctifs de sécurité (6 CVE corrigées) et l'amélioration du code sont des tâches légitimes de maintenance sur un logiciel mature, même si elles causent des régressions temporaires.
- Le mainteneur Tridge est un contributeur OSS extrêmement productif et respecté depuis 30+ ans ; supposer une perte de compétence sans preuves solides serait injuste.
- L'utilisation d'IA pour des tâches bien ciblées et révisées peut être appropriée ; le problème réside dans le manque de révision humaine approfondie des changements générés.
- Les regressions logicielles existaient avant l'IA ; blâmer automatiquement l'IA sans analyse détaillée des causes spécifiques relève du biais confirmation.

**Avis négatifs** :
- Un logiciel mature et stable ne devrait pas subir 26 000 changements en 2 mois sans justification claire ; les risques de régression dépassent les bénéfices pour un outil « terminé ».
- Des régressions fonctionnelles concrètes ont été signalées (copies incrémentielles cassées, augmentation CPU) ; ces problèmes réels justifient la critique indépendamment de la langue utilisée.
- Générer du code sans le comprendre véritablement est une forme de négligence grave sur un logiciel critique pour les sauvegardes et l'infrastructure mondiale.
- La confiance aveugle envers les LLM et l'enthousiasme de « l'effet slot machine » peuvent biaiser le jugement technique, même chez les développeurs expérimentés.
- Les utilisateurs rapportent des problèmes tangibles (fichiers image corrompus lors de copie, défaillances de sauvegardes) ; minimiser ces plaintes comme du manque de preuves ignore l'impact réel.
