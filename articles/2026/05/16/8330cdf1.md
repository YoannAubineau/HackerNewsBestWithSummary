## Résumé de l'article

Radicle est une pile open source peer-to-peer de collaboration sur code construite sur Git, sans entité centrale de contrôle. Les dépôts sont répliqués de manière décentralisée entre les pairs, et les utilisateurs conservent le contrôle total de leurs données et de leur workflow.

- Fonctionne sur Linux, macOS et variantes BSD ; version 1.8.0 disponible avec installation simplifiée par curl
- Utilise des identités cryptographiques pour les artefacts de code, Git pour le transfert de données et un protocole de gossip personnalisé pour les métadonnées
- Les données sociales sont stockées dans Git et signées par cryptographie asymétrique, vérifiées automatiquement
- Offre une expérience locale-first avec fonctionnalité hors ligne, et interface modulaire (CLI, Web, TUI) sur le nœud Radicle
- Les Collaborative Objects (COBs) permettent d'implémenter issues, discussions et revues de code comme objets Git extensibles

## Discussion sur Hacker News (87 commentaires)

**Avis positifs** :
- Radicle offre une alternative souveraine à GitHub en éliminant la dépendance à une entité centralisée et à Microsoft, avec un modèle pair-à-pair natif et des identités cryptographiques.
- L'architecture fondamentale est solide : stocker les issues, patches et métadonnées directement dans Git via des objets CRDT permet une collaboration décentralisée sans serveur central.
- Le projet progresse activement avec des innovations intéressantes comme le support de Jujutsu, les workflows pour agents, l'intégration CI/CD via webhooks, et des travaux sur radicle-artifact pour remplacer Git LFS.
- Pour les petits projets et les clubs tech, Radicle offre une meilleure viabilité à long terme que GitHub (qui pourrait fermer demain) et plus de contrôle sur les données.
- La communauté est engagée, réceptive aux retours, et maintient des outils complémentaires (conteneurs NixOS, documentation d'intégration CI).

**Avis négatifs** :
- L'UX/DX est peu intuitive et peu accessible : pas de création de contenu via web UI, courbe d'apprentissage élevée, navigation confuse entre CLI et explorateur web, bar d'entrée fixée à 10 minutes non atteinte pour les utilisateurs novices.
- La documentation est vague et circulaire (« plateforme collaborative construite sur Git ») sans expliquer clairement quels problèmes elle résout ou comment elle diffère techniquement de Git/GitHub.
- Le déploiement local sans rejoindre le réseau global Radicle est difficile et non documenté : les nœuds de bootstrap se réajoutent automatiquement, création de gestion de réseau complexe pour petits groupes.
- La suppression/redaction de données est structurellement difficile en décentralisé (comme BitTorrent ou Matrix) : fuites accidentelles de secrets ou données sensibles ne peuvent pas être effacées garanties.
- L'absence de licence AGPL (MIT au lieu) risque de permettre à des SAAS de capturer et étendre le projet ; la séparation public/privé repose sur la sélection de réplication plutôt qu'un chiffrement structurel.
