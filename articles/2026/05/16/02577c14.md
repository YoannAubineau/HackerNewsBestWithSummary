## Résumé de l'article

Cet article approfondit les listes HTML au-delà des bases, en présentant les cinq catégories de listes : les listes de contrôle (select, datalist), les listes ordonnées (ol), les listes de description (dl), les menus (menu) et les listes non ordonnées (ul). Chaque type répond à un besoin sémantique distinct basé sur le sens du contenu plutôt que sur son apparence visuelle.

- Les listes de contrôle servent à collecter des données utilisateur : `<select>` pour les choix fixes, `<datalist>` pour les suggestions, avec possibilité d'utiliser `<datalist>` sur n'importe quel type d'input, y compris les curseurs (range)
- Les listes ordonnées (`<ol>`) s'utilisent quand l'ordre change la signification (algorithmes, recettes, séquences), avec les attributs `reversed` (inversion du numérotage) et `start` (numérotation personnalisée)
- Les listes de description (`<dl>`) conviennent aux paires clé-valeur, aux métadonnées et au débogage JSON ; HTML5 permet d'envelopper les termes dans des `<div>` pour les regrouper
- Les menus (`<menu>`) sont réservés aux commandes et contrôles interactifs (barres d'outils), différents des éléments `<nav>` qui sont des sections permettant différents contenus
- Les listes non ordonnées (`<ul>`) constituent le fourre-tout pour les collections où l'ordre n'a pas d'importance

## Discussion sur Hacker News (52 commentaires)

**Avis positifs** :
- L'article offre des informations utiles et souvent méconnues sur les listes HTML, même pour des développeurs expérimentés, avec des exemples pratiques bien testés
- La qualité de l'article contraste agréablement avec beaucoup de contenu de mauvaise qualité actuellement disponible sur le web
- Les éléments HTML natifs comme `<menu>`, `<dialog>` et `<datalist>` offrent des fonctionnalités intéressantes et souvent sous-utilisées par les développeurs
- C'est un bon rappel que les solutions HTML natives simples sont souvent préférables aux solutions React complexes

**Avis négatifs** :
- `<datalist>` a des problèmes de compatibilité majeurs et persistants : ne fonctionne pas bien sur Safari mobile, Firefox Android, et se comporte mal avec les claviers virtuels iOS (GBoard, autofill)
- L'attribut `disabled` sur `<optgroup>` ne fonctionne pas du tout sur Safari iOS malgré une implémentation depuis 2013 sur macOS Safari
- `<datalist>` manque de contrôles suffisants pour être vraiment utile en dehors des prototypes, avec des comportements incohérents (affichage des 3 premiers éléments seulement, non-effacement de l'autofill)
- Il n'existe pas de solution HTML native combinant recherche et sélection, forçant les développeurs à utiliser des composants React même quand HTML serait suffisant
- Les pseudo-classes CSS pour styler les contrôles de formulaires HTML varient entre les navigateurs, rendant React plus fiable malgré sa complexité
