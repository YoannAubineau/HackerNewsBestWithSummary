## Résumé de l'article

Claude Code est un outil d'IA pour la programmation qui fonctionne directement sur les machines des développeurs en naviguant les fichiers localement, sans nécessiter d'index centralisé. Contrairement aux outils basés sur RAG (qui intègrent et indexent le code), Claude Code utilise une recherche agentive qui fonctionne toujours avec le code en direct, évitant les risques de désynchronisation.

- L'écosystème technique autour du modèle (CLAUDE.md files, hooks, skills, plugins, serveurs MCP, intégrations LSP et subagents) détermine les performances plus que le modèle seul
- Les déploiements réussis rendent le code navigable en gardant les fichiers CLAUDE.md épurés et hiérarchisés, en excluant les fichiers générés, et en utilisant des serveurs de protocole de langage pour la navigation par symbole plutôt que par texte
- La configuration Claude doit être maintenue activement tous les trois à six mois car les améliorations des modèles peuvent rendre les instructions anciennes contre-productives
- L'adoption demande une investissement organisationnel : une personne ou une équipe dédiée (infrastructure, développeur expérience, ou gestionnaire d'agent) pour centraliser les conventions, les plugins et la gouvernance
- Les gouvernances anticipées avec groupes de travail interfonctionnels réunissant ingénierie, sécurité informatique et conformité facilitent les déploiements à grande échelle, notamment dans les secteurs régulés

## Discussion sur Hacker News (151 commentaires)

**Avis positifs** :
- Claude Code fonctionne efficacement sur des monorepos massifs et systèmes legacy en production, sans nécessiter d'index centralisé ou d'infrastructure de RAG complexe
- L'approche par traversée de fichiers et grep/find est robuste et fonctionne partout sans configuration préalable, contrairement aux LSP qui demandent de l'entretien
- Avec une bonne configuration (CLAUDE.md, skills, plugins et LSP) et une stratégie de prompt adéquate (planification en amont), les équipes rapportent des gains de productivité réels
- L'intégration avec les outils existants (Jetbrains LSP, scripts custom, MCP) permet d'optimiser la recherche de contexte et réduire la consommation de tokens
- Claude Code excelle à comprendre et documenter de grandes bases de code existantes, et peut corriger rapidement des décisions architecturales isolées et traçables

**Avis négatifs** :
- Claude Code ignore régulièrement les instructions explicites et règles définies dans CLAUDE.md/skills (90% des cas selon certains utilisateurs), rendant la gestion d'une large base de code inefficace
- La consommation de tokens est excessive lors des premières requêtes sur gros projets (jusqu'à 35-50% du quota utilisé d'emblée), car le modèle doit chercher contexte en aveugle par grep répétitif
- Sans index ou LSP, le modèle manque le contexte global et hallucine sur des termes génériques (ex: réimplémente 4 fois un pattern registry au lieu d'en utiliser un existant)
- Claude Code génère du code verbeux avec multiples sources de vérité et duplications (endpoints redondants, migrations non nettoyées) au lieu de refactoriser, rendant nécessaire une révision humaine exhaustive
- L'approche de traversée en première ligne n'est pas celle d'un ingénieur expérimenté : un vrai développeur se souvient du codebase et utilise des outils indexés (CMD+P, LSP); Claude force brute chaque recherche, pollue le context et fait des erreurs
