## Résumé de l'article

Un ingénieur logiciel a construit Fame Boy, un émulateur Game Boy fonctionnel en F# capable de s'exécuter sur desktop et web, après avoir suivi le cours From NAND to Tetris et développé un émulateur CHIP-8. Le projet avait pour objectif principal de mieux comprendre le fonctionnement des ordinateurs au niveau matériel.

- L'architecture sépare l'émulateur en composants (CPU, PPU, APU, contrôleur E/S) qui communiquent via une interface simple composée de deux buffers et deux fonctions, permettant la compatibilité desktop/web.
- F# s'est avéré particulièrement adapté grâce à son système de types robuste : les instructions CPU ont été réduites de 512 opcodes à 58 en modélisant le domaine avec des discriminated unions (From/To), ce qui prévient les états invalides au moment de la compilation.
- L'optimisation de performance a nécessité d'abandonner certains idiomes fonctionnels purs (utilisation de mutabilité et structures plutôt que discriminated unions) : retirer la fonction mapAddress a doublé les FPS en réduisant les allocations heap et les branches JIT.
- Le portage web via Fable a révélé un bug subtil : les opérations bit à bit sur uint8 produisaient des résultats sur 32 bits en JavaScript, nécessitant une troncature manuelle ; le bundle final fait seulement 100 kB.
- L'audio s'est avéré le composant le plus complexe conceptuellement ; l'émulateur utilise l'audio comme horloge pour éviter les pops audio, bien que cette approche soit moins idéale pour la vidéo et nécessite des compromis selon le contexte (mute/non-mute).

## Discussion sur Hacker News (71 commentaires)

**Avis positifs** :
- L'effort humain et l'apprentissage authentique sans LLM sont valorisés, contrastant positivement avec la génération automatisée de code
- F# s'avère bien adapté pour les émulateurs et projets d'apprentissage complexes, permettant de bonnes abstractions fonctionnelles tout en acceptant l'impératif quand nécessaire
- La modélisation des données en F# (discriminated unions, type systems) offre une clarté et une expressivité impressionnantes par rapport à des langages impératifs
- Les émulateurs sont reconnus comme excellents projets pédagogiques pour apprendre un nouveau langage, particulièrement pertinent pour explorer différents paradigmes de programmation

**Avis négatifs** :
- F# reste prisonnier de l'écosystème C#/.NET : les bibliothèques sont orientées OOP, les documentations manquent de spécificité F#, obligeant à des compromis hybrides OOP/FP
- La performance du code F# idéomatique (structures immuables, listes chaînées) reste inférieure au code impératif ; les vrais projets exigent souvent d'abandonner la beauté fonctionnelle pour des boucles impératives mutable
- Fable a des comportements non-standards problématiques (conversion JavaScript Number élargissant les uint8), rendant la transpilation web naive et peu fiable
- La courbe d'adoption reste difficile : beaucoup associent F# à une période obsolète (.NET Framework), et l'absence de documentation spécifique crée des barrières d'apprentissage
