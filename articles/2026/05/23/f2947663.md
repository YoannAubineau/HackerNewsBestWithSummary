## Résumé de l'article

Microsoft retire progressivement l'accès à Claude Code, l'outil de codage d'Anthropic, pour ses développeurs et les rediriger vers GitHub Copilot CLI, sa propre solution de ligne de commande. Bien que Claude Code se soit avéré populaire en interne ces six derniers mois, Microsoft souhaite converger sur son propre outil et réduire les dépenses d'exploitation.

- L'équipe Experiences + Devices de Microsoft (Windows, Microsoft 365, Outlook, Teams, Surface) cessera d'utiliser Claude Code fin juin 2024, dernier jour de l'exercice fiscal
- Les développeurs sont encouragés à migrer leurs workflows vers Copilot CLI dès les prochaines semaines, bien que Claude Code ait surpassé Copilot CLI en adoption interne
- Microsoft investira davantage dans Copilot CLI pour l'intégrer profondément dans ses workflows d'ingénierie ; les modèles d'Anthropic resteront accessibles via Copilot CLI
- La décision est à la fois stratégique (converger sur un outil propriétaire) et financière (économies réalisées au changement d'exercice fiscal)
- Les modèles Claude d'Anthropic demeurent privilégiés par Microsoft dans Microsoft 365 et Copilot pour certaines tâches, et le partenariat Azure Foundry n'est pas affecté

## Discussion sur Hacker News (226 commentaires)

**Avis positifs** :
- Microsoft devrait enfin « manger son propre chien » en utilisant ses propres outils plutôt que de dépendre d'Anthropic, ce qui pourrait accélérer l'amélioration de Copilot
- Une utilisation responsable des tokens existe avec des workflows supervisés et orientés humain, contrairement aux approches agentic non contrôlées qui brûlent les ressources
- Les coûts imprévisibles des APIs LLM sont un problème réel pour les entreprises, justifiant une transition vers des solutions plus contrôlables et moins coûteuses
- Le problème révèle l'insoutenabilité économique actuelle : même les géants comme Microsoft trouvent les modèles frontier trop chers, confirmant que les pricing actuels ne sont pas viables

**Avis négatifs** :
- Le titre est trompeur : il s'agit simplement d'une migration vers Copilot, pas d'une réduction globale des dépenses IA, Microsoft n'ayant pas de modèles concurrents propres
- Les employés de Microsoft auraient préféré Claude Code à Copilot CLI selon les utilisateurs, suggérant que la décision est surtout politique et commerciale plutôt que basée sur la qualité
- Copilot CLI serait nettement inférieur aux modèles frontier actuels (1-2 ans de retard), ce qui pourrait dégrader la productivité des développeurs malgré les économies d'échelle
- La vraie cause est probablement le coût exorbitant lié à des usages inefficaces et non supervisés lors du pilot, pas un problème fondamental avec Claude Code lui-même
- Le problème des coûts AI imprévisibles reste sans solution : les entreprises plus petites n'ont pas les ressources pour auto-héberger et seront exclues de l'utilisation efficace des outils IA
