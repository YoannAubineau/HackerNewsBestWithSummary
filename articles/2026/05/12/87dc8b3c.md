## Discussion sur Hacker News (974 commentaires)

**Avis positifs** :
- Statically typed languages like Rust and Go provide better feedback loops for LLMs through compile-time errors, enabling faster iteration and fewer runtime failures compared to Python's dynamic typing.
- Rust's strict type system and memory safety guarantees reduce the class of bugs LLMs can introduce, making generated code more reliable without extensive manual review.
- Performance gains are substantial in practice: multiple commenters report 10x-100x speedups when porting from Python to compiled languages like Rust or Go, with smaller binary sizes.
- Languages like Go offer simplicity and consistency that naturally guide LLMs toward idiomatic code, reducing the need for extensive prompting and preventing code bloat.
- The ecosystem shift means Python libraries increasingly wrap Rust/C++ implementations anyway, so using Rust directly eliminates unnecessary indirection and improves deployment characteristics.

**Avis négatifs** :
- Python remains superior for code review by humans who need to understand and maintain the code; readability and familiarity reduce cognitive load when verifying AI output.
- Python's ML/AI ecosystem (NumPy, PyTorch, scikit-learn) is unmatched and deeply integrated; using other languages requires rewriting or creating bindings, negating efficiency gains.
- Rust's longer compile times and higher verbosity consume more tokens and slow the iteration loop, potentially offsetting gains from fewer runtime errors.
- Developers still need to read, understand, and debug AI-generated code in any language; choosing one they don't know well creates a maintenance liability if the AI can't fix complex issues.
- Empirical evidence is mixed: AutoCodeBenchmark and other studies show LLM performance varies significantly by language independent of typing (Elixir performs well despite being dynamic), suggesting language choice matters less than code quality in training data.
