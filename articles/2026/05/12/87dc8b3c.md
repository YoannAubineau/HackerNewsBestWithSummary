## Discussion sur Hacker News (306 commentaires)

**Avis positifs** :
- Les LLM écrivent du code plus fiable en langages typés et compilés (Rust, Go) car les erreurs de compilation rattrapent les hallucinations du modèle, contrairement à Python où les bugs peuvent rester cachés jusqu'à l'exécution.
- La verbosité de Rust force le LLM à être explicite sur les détails critiques (gestion mémoire, concurrence), réduisant les chemins d'erreur possibles et les bugs subtils que les tests pourraient ne pas capturer.
- Rust et Go produisent du code plus simple à relire et vérifier pour un humain, même sans expertise préalable, grâce à la clarté imposée par le système de types et le compilateur.
- Les langages compilés offrent une meilleure performance (10-100x) sans coût additionnel une fois le code généré, ce qui devient pertinent pour les applications réelles à l'échelle.
- Go est particulièrement bien adapté car il combine forte typage, cohérence de code (peu de façons de faire), compilation rapide et apprentissage rapide même pour les LLM.

**Avis négatifs** :
- Apprendre Rust ou Go pour maintenir du code AI-généré ajoute une charge cognitive importante si l'équipe connaît déjà Python, et le coût d'apprentissage peut dépasser les bénéfices pour de petits projets.
- Python reste plus lisible pour les humains et plus facile à déboguer rapidement en cas de panne, ce qui compte lors de crises 3h du matin sans accès aux LLM.
- Les LLM produisent réellement du code de meilleure qualité en Python selon certains benchmarks (SWE-Bench), et bénéficient d'une gigantesque base de code d'entraînement dont l'écosystème ML est incomparable.
- La majorité des équipes ne vont jamais à la vraie production sans review humain, donc la lisibilité pour les humains reste critique ; miser sur du code 'vibe-coded' en Rust sans expertise est risqué.
- Pour les prototypes rapides, les scripts one-shot et les projets exploratoires, Python et TypeScript conservent un avantage substantiel grâce à l'absence de compilation et aux itérations plus rapides.
