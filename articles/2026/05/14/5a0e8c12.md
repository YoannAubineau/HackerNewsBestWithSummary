## Résumé de l'article

Le chercheur en sécurité Chaotic Eclipse a publié deux nouvelles vulnérabilités critiques affectant Windows : YellowKey, un exploit BitLocker permettant d'accéder à un disque chiffré sans clé, et GreenPlasma, une faille d'escalade de privilèges. Ces divulgations interviennent après que Microsoft aurait rejeté ses rapports de vulnérabilité antérieurs.

- YellowKey permet de contourner le chiffrement BitLocker en copiant des fichiers sur une clé USB et en redémarrant dans l'environnement de récupération Windows, sans nécessiter les clés de chiffrement
- L'exploit fonctionne sur Windows 11 et Windows Server 2022/2025, mais non sur Windows 10 ; il contournerait même les configurations TPM+PIN selon le chercheur
- GreenPlasma vise une escalade de privilèges locale via la manipulation du processus CTFMon pour accéder à des sections mémoire partagées et obtenir un accès au niveau SYSTEM
- BitLocker protège des millions d'appareils mondiaux (activé par défaut sur Windows 11), rendant ces exploits particulièrement graves pour les entreprises et gouvernements
- Microsoft n'a pas encore réagi officiellement aux divulgations YellowKey et GreenPlasma ; BlueHammer a déjà été corrigé

## Discussion sur Hacker News (127 commentaires)

**Avis positifs** :
- L'exploit YellowKey démontre une vulnérabilité sérieuse et récurrente : le chercheur a découvert 5 zéro-days en 3 mois, suggérant des problèmes systémiques profonds dans BitLocker plutôt que des bugs isolés.
- Microsoft a historiquement traité BitLocker légèrement : depuis Windows 7, des failles permettaient de contourner le chiffrement via le CD d'installation, révélant une inattention chronique à la sécurité du démarrage.
- L'exploit repose sur des mécanismes spécifiques (noms de fichiers déclenchant le déchiffrement automatique, suppression de traces) qui ressemblent trop précisément à une porte dérobée intentionnelle pour être une simple mauvaise implémentation.
- La conception par défaut de BitLocker (clé dans le TPM sans PIN, déchiffrement automatique au boot) est fondamentalement défectueuse : protéger un disque chiffré avec une clé stockée localement sur le même appareil offre peu plus de protection qu'aucun chiffrement.
- Les silences de Microsoft : les exploits RedSun et Bluehammer ont été corrigés discrètement sans reconnaissance du chercheur ni réponse aux CVE, révélant une stratégie de dissimulation plutôt que de transparence.

**Avis négatifs** :
- Il n'existe pas de preuve concrète que ceci est volontaire : le bug pourrait être une mauvaise gestion de transactions de système de fichiers ou une simple incompétence, non une porte dérobée intentionnelle, ce qu'une confirmation directe de Microsoft ne clarifierait jamais.
- L'exploit nécessite un démarrage via WinRE (Environnement de Récupération Windows) et affecte uniquement les modes TPM seul sans PIN : c'est une faille d'authentification post-boot, pas un vrai contournement du chiffrement BitLocker lui-même.
- BitLocker ne fonctionne pas différemment de Linux/LUKS avec déchiffrement automatique scellé au TPM : tout OS utilisant l'identité matérielle pour déverrouiller le disque est vulnérable aux mêmes attaques post-boot, ce problème n'est pas unique à Microsoft.
- L'auteur affirme avoir bypassé TPM+PIN mais refuse de le démontrer publiquement : sans preuve concrète, l'affirmation d'un 6e exploit après 5 zéro-days reste extraordinaire et demande une vérification indépendante.
- Les PIN de BitLocker sont bien entrelacés avec le matériel clé du TPM avec protections anti-brute-force : même si la clé peut être extraite du TPM, le PIN reste nécessaire pour y accéder, ce qui n'est pas aussi simple que présenté.
