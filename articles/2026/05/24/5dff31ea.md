## Résumé de l'article

Microsoft a publié cette semaine le code source de 86-DOS 1.00, le plus ancien code DOS connu à ce jour, antérieur même au branding MS-DOS. Cette version inclut des noyaux de développement de PC-DOS 1.00 et des utilitaires historiques comme CHKDSK.

- Le code provient de 86-DOS, créé par Tim Paterson pour l'ordinateur Intel 8086 de Seattle Computer Products, sous le nom initial QDOS (« quick and dirty operating system »)
- Microsoft a acquis les droits de 86-DOS pour développer un système d'exploitation destiné à l'IBM PC 5150 en cours de développement
- Microsoft a concédé la version PC-DOS à IBM tout en vendant sa propre version, MS-DOS, à d'autres constructeurs
- MS-DOS est devenu dominant auprès du public grâce à la prolifération des clones IBM PC dans les années 1980 et 1990

## Discussion sur Hacker News (61 commentaires)

**Avis positifs** :
- Publication historique majeure : le code source de DOS révèle les décisions d'architecture et offre un document historique sur l'informatique des années 1980
- Valeur pédagogique : permet de comprendre comment concevoir efficacement avec des ressources limitées et de retrouver une compréhension des fondamentaux informatiques souvent perdus chez les développeurs modernes
- Exploit archivistique remarquable : des historiens ont réussi à numériser et transcrire le code à partir de vieux listings papier, démontrant la durabilité du papier face aux supports numériques disparus
- Ouverture encourageante : Microsoft a également publié le BASIC source code, signalant une tendance positive vers l'open-sourcing de code legacy et historique

**Avis négatifs** :
- Intérêt limité en 2026 : le code est arrivé trop tard pour avoir un impact pratique significatif ; DOS est obsolète depuis des décennies
- Espoirs déçus pour Windows : bien que DOS soit publié, les versions anciennes de Windows (2000, 95, 3.11) restent fermées malgré les fuites partielles, créant une asymétrie frustrante
- Risques légaux pour les projets de reconstruction : utiliser ce code ou des LLM pour recréer Windows via des outils comme ReactOS présente d'importants problèmes juridiques et de propriété intellectuelle
- Utilité pratique décroissante : les commentateurs questionnent les cas d'usage réels en 2026 pour du code DOS vieux de 45 ans, au-delà de l'intérêt archivistique
