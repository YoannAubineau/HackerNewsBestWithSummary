## Discussion sur Hacker News (172 commentaires)

**Avis positifs** :
- HTML offre des capacités bien supérieures à Markdown pour les tableaux, visualisations SVG, interactions JavaScript et design personnalisé, permettant des documents plus riches et engageants.
- Les fichiers HTML single-file sont faciles à partager par email ou lien, ouvrent nativement dans n'importe quel navigateur, et permettent une linkabilité complète essentielle pour la collaboration interne.
- Les LLMs génèrent du HTML de manière fiable et sophistiquée ; créer une page HTML simple à partir d'une requête est devenu un workflow extrêmement productif pour prototyper rapidement des outils et dashboards.
- HTML reste suffisamment lisible et éditable à la main ; contrairement aux idées reçues, les balises de base sont simples à comprendre et les éditeurs modernes offrent une excellente support.
- Pour les cas d'usage interactifs (rapports, spécifications dynamiques, prototypes), HTML permet une meilleure expérience utilisateur et une compression d'information plus efficace que du Markdown verbeux.

**Avis négatifs** :
- HTML consomme 2 à 4 fois plus de tokens que Markdown pour exprimer le même contenu structurel, augmentant les coûts d'inférence de manière significative.
- La co-édition humaine-agent devient beaucoup plus difficile avec HTML ; modifier un document généré requiert souvent de repromter l'agent plutôt que d'éditer directement, bloquant le flux de travail créatif.
- Markdown inline supporte déjà l'HTML pour les cas avancés, et GitHub Markdown offre des fonctionnalités étendues (diagrammes Mermaid, etc.) ; HTML général n'est pas toujours nécessaire.
- La préoccupation légitime que cette tendance crée une dépendance accrue aux agents pour maintenir les documents, réduisant l'autonomie des utilisateurs et favorisant une délégation complète plutôt que la collaboration.
- HTML manque d'une parsabilité structurelle et de vérifiabilité programmatique (contrairement à JSON/XML) ; formats structurés seraient préférables pour l'analyse statique et la validation des spécifications.
