## Discussion sur Hacker News (253 commentaires)

**Avis positifs** :
- Le test à 99,8% de compatibilité sur Linux x64 glibc démontre que les LLM excellent à traduire du code entre langages, particulièrement quand une suite de tests complète et de qualité existe pour vérifier le résultat
- Rust offre des garanties de sécurité mémoire (borrow checker, pas de use-after-free, pas de buffer overruns) que Zig n'a pas, ce qui devrait réduire drastiquement les fuites mémoire et crashs qui affligeaient Bun
- L'exploit technique démontre une capacité impressionnante : 960 000 lignes de code portées en 6 jours avec un taux de succès quasi complet, ce qui aurait pris des mois à une équipe humaine
- La récompilation est aussi rapide qu'en Zig grâce au compilateur Zig plus rapide utilisé lors du développement initial, et le code Rust compilé devrait être aussi performant que l'original
- La portage d'un projet existant avec suite de tests complète et bonne architecture est exactement le cas d'usage où les LLM performent le mieux, contrairement aux réécritures ex nihilo

**Avis négatifs** :
- 14 000 blocs `unsafe` générés par le LLM posent question sur la maintenabilité réelle et si les avantages de sécurité de Rust sont vraiment conservés ; il faudra des mois de travail supplémentaire pour les éliminer
- Le code généré à cette vitesse ne peut pas avoir été revu en profondeur : personne ne comprend vraiment ce qui a été construit, rendant la maintenance future et l'évolution difficiles sans risquer de refactorings en cascade
- Cet exploit repose entièrement sur les infrastructures massives et les tokens illimités d'Anthropic ; les entreprises sans ce capital ne peuvent pas reproduire cet exploit, creusant un fossé entre ceux qui ont et les autres
- Bun, ancien porte-étendard de Zig, abandonne maintenant le langage, ce qui envoie un mauvais signal : même ses plus grands projets jugent Zig insuffisant, menaçant la viabilité future du langage encore en bêta
- L'affirmation que cela aurait pris « des centaines d'ingénieurs plus d'un an » est exagérée et trompeuse ; un petit nombre de développeurs Rust compétents auraient probablement achevé cette traduction bien plus vite avec la même suite de tests
