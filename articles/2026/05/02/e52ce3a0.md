## Résumé de l'article

OpenWarp est une extension du terminal Warp qui introduit la capacité BYOP (Bring Your Own Provider) permettant d'utiliser six protocoles API natifs différents. Elle conserve toutes les fonctionnalités d'interaction de Warp tout en donnant aux utilisateurs le contrôle complet sur les clés API, les modèles et les invites système.

- Support natif de six protocoles API : OpenAI, Anthropic, Gemini, Ollama et DeepSeek peuvent être configurés librement avec des URL de base et clés API personnalisées
- Les données d'authentification restent stockées localement et ne sont jamais envoyées au cloud, Cloud Agent et Computer Use étant désactivés par défaut
- Moteur de modèles basé sur minijinja pour adapter dynamiquement les invites système en fonction du répertoire actif, de la langue et du rôle utilisateur
- Interface utilisateur native en chinois simplifié, anglais, japonais et espagnol, extensible par la communauté
- Compatibilité complète avec l'amont Warp incluant les blocks, workflows, commandes AI, mappages de touches et thèmes, avec licence double AGPL-3.0/MIT

## Discussion sur Hacker News (134 commentaires)

**Avis positifs** :
- OpenWarp répond à un besoin réel : les utilisateurs veulent Warp comme terminal sans les features AI intrusives et sans obligation de compte payant
- Forking et créer ses propres versions est au cœur même de l'open source ; c'est un droit légitime pour ajouter/retirer des fonctionnalités
- Warp Inc. a déçu sa communauté en pivotant vers l'IA/agent au lieu de maintenir les améliorations UI terminales apportées initialement (blocs, tabs, prédictions)
- Le projet offre une vraie alternative pour ceux bloqués par le paywall Warp ou les exigences de création de compte pour utiliser ses propres modèles
- C'est un signal fort aux entreprises : ignorer les retours utilisateurs et monétiser à outrance risque une révolte communautaire

**Avis négatifs** :
- Réutiliser le nom 'Warp' est déceptif, contraire aux usages éthiques du libre, et constitue potentiellement une violation de marque déposée (USPTO #90342558)
- C'est une fork prématurée : le projet a été ouvert il y a 24h seulement, il n'existe pas encore de vraie communauté de développeurs ayant raison de se séparer
- Le fondateur Warp s'engage déjà à ajouter le BYOM (bring-your-own-model) directement dans Warp, rendant le fork obsolète avant même de démarrer
- OpenWarp ne se différencie pas vraiment : il requiert toujours un compte payant pour utiliser ses propres fournisseurs (sur la branche master)
- Manquer de décence en réutilisant le même nom dilue la marque et trompe les utilisateurs ; un vrai fork respectueux changerait de nom et engagerait d'abord avec le projet
