## Résumé de l'article

Visual Studio Code, l'éditeur de code de Microsoft, insérerait automatiquement une ligne de co-création attribuant les commits à GitHub Copilot, même lorsque cet outil d'IA n'a pas été utilisé.

- VS Code ajoute une ligne « Co-Authored-by Copilot » aux messages de commit de manière systématique
- Ce comportement se produit indépendamment de l'utilisation réelle de Copilot pour écrire le code
- Cette pratique soulève des questions sur l'attribution correcte des contributions et la transparence des outils utilisés
- Les utilisateurs ne semblent pas pouvoir contrôler facilement ce comportement automatique

## Discussion sur Hacker News (137 commentaires)

**Avis positifs** :
- L'ajout de co-auteur peut servir de métadonnée utile pour identifier et tracker le code généré par IA, similar aux métadonnées EXIF des photos ou PDF
- Certains considèrent que c'est une forme de transparence légitime si l'IA a réellement contribué au code, comparable à créditer un compilateur pour ses optimisations
- Claude et d'autres outils font déjà la même chose depuis longtemps, ce ne serait donc pas une nouveauté absolue de l'écosystème

**Avis négatifs** :
- VSCode ajoute le tag même quand Copilot n'a jamais été utilisé ou que toutes les fonctionnalités IA sont désactivées—c'est de la fausse attribution à des fins de marketing
- Falsifier l'autorship dans les commits crée des registres techniques et légaux inexacts, et pourrait impacter les droits d'auteur (le code généré par IA n'étant pas copyrightable, cela pourrait invalider la protection)
- Cela représente du co-autorship forcé sans consentement pour servir les métriques vanité de Microsoft et gonfler les stats d'utilisation de Copilot, transformant les développeurs en canal publicitaire non rémunéré
- C'est un comportement d'enshittification typique: Microsoft a construit VSCode en produit gratuit de qualité, puis l'utilise comme vecteur pour imposer des pratiques contraires à l'intérêt des utilisateurs
- Le changement a été fusionné sans explication adéquate (aucune description), avec des débuts de passage off→on→chatAndAgent, et même Copilot avait signalé l'incohérence du code en revue
- Cela montre une hostilité systémique envers les standards et l'UX: après des décennies de réhabilitation, Microsoft sacrifie sa réputation pour la ruée vers l'IA
