## Résumé de l'article

Python 3.15 introduit plusieurs améliorations utiles au-delà des grandes annonces. Parmi elles figurent des outils pour la programmation asynchrone, les décorateurs, la sécurité des threads et des opérations sur les structures de données.

- **Annulation gracieuse de TaskGroup** : nouvelle méthode `TaskGroup.cancel()` simplifiant l'interruption des tâches asynchrones sans lever d'exceptions manuellement
- **Décorateurs améliorés avec les gestionnaires de contexte** : `ContextDecorator` détecte maintenant le type de fonction décorée (asynchrone, générateur, etc.) pour appliquer correctement le contexte sur leur cycle de vie entier
- **Itérateurs thread-safe** : nouvelles utilitaires `threading.serialize_iterator`, `threading.synchronized_iterator` et `threading.concurrent_tee` permettant l'utilisation sécurisée des itérateurs dans du code multi-thread
- **Opération XOR pour Counter** : `collections.Counter` supporte maintenant l'opérateur `^` pour la différence symétrique entre compteurs
- **Objets JSON immuables** : `json.loads` accepte un paramètre `array_hook` permettant de créer des structures JSON entièrement immuables (hashables) avec tuples et `frozendict`

## Discussion sur Hacker News (162 commentaires)

**Avis positifs** :
- Les améliorations des messages d'erreur (suggestions cross-langage) et des primitives de synchronisation pour itérateurs sont des ajouts pratiques et bienvenus.
- Les décorateurs ContextDecorator supportant async functions/iterators comblent enfin un problème de longue date, même si le changement de comportement était peu utilisé auparavant.
- Les imports lazy et l'amélioration des ExceptionGroup simplifient certains patterns courants (chargement conditionnel, gestion d'erreurs multiples en async).
- Python reste dominant pour l'IA/ML grâce à son écosystème (numpy, pandas) et reste viable pour le prototypage rapide malgré ses limitations de performance.

**Avis négatifs** :
- Python perd progressivement ses utilisateurs au profit de Go, Rust et TypeScript pour les nouveaux projets, surtout en backend et systèmes, en raison de sa performance lente (50-200x plus lent que C++/Rust).
- La multiplication de features (lazy imports, type hints, decorators avancés) complexifie le langage et le dilue de sa philosophie d'origine ; les imports lazy sont perçus comme un pansement sur une architecture mal conçue.
- Les problèmes structurels (GIL, concurrence, indentation comme sémantique) ne sont pas résolus ; les données d'entraînement pour l'IA contiennent beaucoup de Python 2 ou de code de faible qualité, limitant les gains attendus.
- L'écosystème Python souffre d'inconsistance et d'attaques de chaîne d'approvisionnement dues au modèle pip/PyPI ; les risques de sécurité ne sont pas adéquatement adressés.
- Les derniers changements semblent dictés par de gros utilisateurs corporatifs (Instagram/Meta) plutôt que par les besoins de la communauté générale, dénaturant le caractère du langage.
