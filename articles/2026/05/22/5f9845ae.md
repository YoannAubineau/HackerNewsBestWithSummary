## Résumé de l'article

Un développeur a construit un système d'indexation locale pour cataloguer une année de vidéos brutes d'un lodge en Afrique, utilisant Gemma 4 31B sur son MacBook M1 Max avec 64GB de RAM et jusqu'à 50GB de swap. Le projet résout le problème en amont : rendre les archives vidéo interrogeables en langage naturel, plutôt que d'utiliser directement des éditeurs IA qui supposent les fichiers étiquetés.

- Le pipeline extrait métadonnées, géolocalisation, transcription avec diarisation (WhisperX), détection faciale (InsightFace) et génération de descriptions via vision model, produisant des fichiers sidecars Markdown à côté de chaque clip.
- Trois backends vision sont intégrés : Claude CLI (sans coût supplémentaire), API Anthropic (pour la vitesse), et LM Studio local (pour les traitements en masse) ; Gemma 4 31B produit des résultats comparables à Claude Sonnet 4.6 pour 80% des cas.
- Quatre bogues démontrés révèlent des leçons : les schémas énumérés préviennent mieux les hallucinations que les instructions libres, les permissions silencieuses de CLI demandent des contrôles explicites, éviter les champs union-type en schéma, et différencier les critères de tri selon le contexte (archives photo vs mémoires vidéo).
- Le coût était un weekend de temps et 50GB de swap ; l'indexeur résout la moitié du problème (trouver le bon clip), l'éditeur piloté par Claude Code (avec DaVinci Resolve MCP et ElevenLabs) est la construction suivante.

## Discussion sur Hacker News (93 commentaires)

**Avis positifs** :
- L'exploit technique est impressionnant : faire tourner localement Gemma 4 31B sur une MacBook M1/M5 avec indexation vidéo complète démontre le potentiel réel des modèles locaux pour les tâches personnelles intensives
- L'approche combine intelligemment des outils spécialisés (insightface pour les visages, exiftool pour les métadonnées GPS, LLM pour la description) plutôt que de demander à l'IA généraliste de tout faire, résolvant les limitations intrinsèques des modèles
- La bande passante mémoire d'Apple Silicon rend viable ce qui serait inutilisable sur x86, ouvrant des cas d'usage jusqu'alors impossibles sur ordinateur personnel
- L'absence de limites API et les préoccupations de confidentialité/fiabilité/restrictions rendent les modèles locaux attrayants pour traiter de grandes quantités de données personnelles sensibles
- L'open-sourcing du projet et la disposition à collaborer créent une valeur communautaire réelle pour d'autres avec des besoins similaires d'archivage personnel

**Avis négatifs** :
- Le texte souffre de marqueurs d'IA généraliste perceptibles (énumération excessive, déconnexion entre les phrases, tournures maladroites) qui nuisent à la clarté et la crédibilité, même avec le contenu technique solide
- L'utilisation de 50 GB de swap accélère l'usure du SSD et les applications Electron simultanées causent probablement la majorité de la thrashing ; des optimisations logicielles pourraient réduire drastiquement ce besoin
- Le coût effectif reste prohibitif pour les utilisateurs cloud (1 dollar par heure de vidéo analysée avec Claude), bien que l'auteur évite ce problème localement
- La sélection assistée d'IA des meilleures photos reste un problème difficile car le goût artistique est subjectif et il y a risque de rejeter les meilleurs clichés, limitant l'utilité pratique
