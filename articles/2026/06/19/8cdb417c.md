## Résumé de l'article

Project Valhalla est un projet Java de dix ans qui vise à permettre aux programmeurs d'écrire des classes performantes et compactes en mémoire, sans sacrifier la lisibilité. En juin 2025, Oracle a confirmé l'intégration de la JEP 401 (Value Classes and Objects) dans JDK 28, prévue pour mars 2027, bien qu'en tant que preview et désactivée par défaut.

- Une value class se déclare avec le modificateur `value` et crée des objets sans identité, contrairement aux classes ordinaires : deux instances identiques sont considérées comme équivalentes plutôt que distinctes (l'opérateur == teste la substitutabilité, non l'adresse mémoire).
- L'intérêt principal est la densité mémoire : au lieu de stocker un million de pointeurs vers des objets dispersés sur le tas, une `Point[]` peut maintenant stocker des valeurs compactes côte à côte, exploitant l'efficacité du cache processeur.
- Deux techniques d'optimisation permettent cette densité : la scalarization (le compilateur JIT décompose les références en champs bruts, sans allocation) et le heap flattening (les données sont encodées directement dans les tableaux ou champs, sans pointeur indirection).
- La non-nullabilité reste optionnelle en JDK 28 : une value class peut toujours être null, car c'est un type référence. Les types null-restreints constituent une JEP séparée et future, clé pour l'aplatissement des classes plus grandes.
- Les génériques spécialisés (permettant `ArrayList<Point>` vraiment plat) ne sont pas encore inclus : JDK 28 n'apporte que l'aplatissement pour les tableaux et champs directs, pas pour les collections génériques, car le type erasure persiste.
