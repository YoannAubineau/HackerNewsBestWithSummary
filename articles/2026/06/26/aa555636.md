## Résumé de l'article

OpenKnowledge est un éditeur markdown local et wiki alimenté par IA qui fonctionne comme alternative open source à Obsidian et Notion. Il propose une interface WYSIWYG, des intégrations avec Claude et d'autres modèles de langage, et est disponible en application macOS ou en application web pour Linux, Windows et Intel Mac.

- Éditeur WYSIWYG pour éditer des fichiers markdown avec une expérience similaire à Google Docs ou Notion
- Intégrations IA natives avec Claude, Codex et support des applications Cursor via protocole MCP et CLI
- Partage d'équipe sans code et synchronisation automatique utilisant git/GitHub en arrière-plan
- Installation via application DMG sur macOS ou via CLI Node.js (v24+) sur Linux/Windows/Intel Mac
- Licence GPL-3.0-or-later avec contributions publiques acceptées et automatiquement mirrorées dans le dépôt interne

## Discussion sur Hacker News (126 commentaires)

**Avis positifs** :
- Excellente alternative pour migrer d'Obsidian vers une solution open source, avec support de la syntaxe Obsidian (wikilinks) et import direct des vaults Markdown
- Éditeur WYSIWYG de niveau Notion (drag-and-drop, slash commands) qui manque aux solutions Markdown pures, combiné avec intégration native MCP/skills pour les agents IA
- Architecture locale et versionnée via Git/GitHub, évitant le vendor lock-in tout en facilitant la collaboration d'équipe et la synchronisation sans dépendance cloud
- CLI multi-plateforme et web viewer permettant aux agents IA (Claude, Codex) d'ouvrir l'éditeur dans leurs propres interfaces, améliorant l'expérience agentic
- Recherche sémantique et virtualisation des commandes système intégrées pour les agents, offrant une meilleure navigation hiérarchique des documents

**Avis négatifs** :
- Réduction d'Obsidian à 'juste du Markdown' omet les fonctionnalités avancées comme dataview et les dashboards dynamiques—lacunes significatives pour utilisateurs expérimentés
- Intégration IA limitée aux services propriétaires (Claude, Codex, Cursor) sans support initial pour modèles locaux, contredisant le positionnement 'open source'
- Synchronisation basée sur GitHub uniquement crée des friction sur les images/attachments volumineux (problèmes de git-lfs) et semble orientée équipes plutôt que utilisateurs individuels multi-appareils
- Affichage trompeur sur le site web montrant des capacités de chat dans l'app alors qu'elles ne fonctionnent que via débordement vers des agents externes, créant une déception utilisateur
- Performance et expérience utilisateur déficitaires par rapport à Obsidian/VS Code (app Electron lente, manque de finition, absence de système de plugins comparable)
