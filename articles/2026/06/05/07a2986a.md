## Résumé de l'article

Anthropic publie une implémentation de référence open source permettant de découvrir et corriger automatiquement les vulnérabilités de sécurité en utilisant Claude. Cet outil s'appuie sur l'expérience acquise lors de partenariats avec des équipes de sécurité, et propose une boucle autonome : reconnaissance → détection → vérification → rapport → correctif.

- Le framework inclut des outils interactifs (Claude Code skills) pour construire des modèles de menace, scanner les vulnérabilités, les trier et générer des correctifs candidats, tous fonctionnant en lecture/écriture de fichiers sans isolation requise
- Le pipeline autonome s'exécute dans des conteneurs gVisor isolés et utilise plusieurs agents spécialisés : un agent de reconnaissance qui partitionne le code, des agents de détection en parallèle qui créent des entrées malformées, un vérificateur, un dédoublonneur et un générateur de rapports
- L'approche recommandée commence par des tâches interactives le jour 1 (modèle de menace, scan statique, triage, correctifs), puis passe à des exécutions autonomes le jour 2 sur des cibles connues, avant de customiser le pipeline pour ses propres projets
- Le framework est conçu initialement pour détecter les vulnérabilités mémoire en C/C++ via ASAN, mais sa structure générique peut être portée à d'autres langages et classes de vulnérabilités en répondant à des questions de configuration
- Anthropic propose également Claude Security, un produit hébergé pour la détection et correction des vulnérabilités, tandis que ce référentiel n'est pas maintenu et n'accepte pas de contributions

## Discussion sur Hacker News (86 commentaires)

**Avis positifs** :
- La détection de vulnérabilités par IA est un excellent cas d'usage car elle consiste principalement à faire du pattern matching sur du code précis et analysable
- Comparé aux audits de sécurité formels traditionnels, le coût peut être très avantageux même s'il y a un investissement initial important en tokens
- L'approche des 'shop jigs' personnalisés est pertinente : chacun peut adapter l'outil à son propre flux de travail plutôt que de dépendre d'une solution générique
- Les modèles comme Mythos et Claude trouvent effectivement des vulnérabilités sérieuses dans le code hérité que les outils traditionnels et les révisions PR ne détectent pas

**Avis négatifs** :
- Le modèle économique est problématique : plus on cherche (plus on paie), plus on a de chances de trouver quelque chose, ce qui ressemble à un 'loot box' plutôt qu'une solution fiable
- Le coût d'exécution est prohibitif pour une utilisation continue : plusieurs millions de dollars annuels par entreprise selon les calculateurs disponibles
- Sans excellente gouvernance, l'outil produit trop de faux positifs ('vibe auditing') qui fatiguent les développeurs plutôt que de vraiment les aider
- L'outil n'est pas maintenu et n'accepte pas les contributions, il sert principalement de vitrine pour pousser vers les offres commerciales payantes d'Anthropic
