## Résumé de l'article

Open Code Review est un outil de révision de code en ligne de commande, alimenté par IA, développé initialement par Alibaba Group. Il combine une architecture déterministe avec un agent IA pour analyser les diffs Git, identifier les défauts de code avec précision au niveau des lignes, et générer des commentaires structurés.

- Combine une ingénierie déterministe (sélection précise des fichiers, regroupement intelligent, correspondance des règles) avec un agent IA pour des décisions dynamiques et une récupération contextuelle
- Résout les limites des outils génériques : couverture complète des changements, positionnement exact des problèmes, qualité stable indépendante des variations de prompt
- S'installe via NPM ou binaires GitHub, nécessite la configuration d'un endpoint LLM (Claude, Anthropic, etc.) et supporte les modes interactif, branche et commit
- Intégration complète : slash commands dans Claude Code, plugins de compétences (skills), pipelines CI/CD (GitHub Actions, GitLab CI), visualiseur de sessions
- Offre des commandes avancées : prévisualisation sans appels LLM, concurrence configurable, règles de révision personnalisées, observabilité OpenTelemetry, historique des sessions

## Discussion sur Hacker News (66 commentaires)

**Avis positifs** :
- Résout un besoin réel : la revue de code est un goulot d'étranglement, et l'automatisation IA peut apporter une valeur significative en complément des revues humaines
- Approche hybride innovante combinant déterminisme (filtrage de fichiers, regroupement intelligent) et agents IA, réduisant le bruit contextuel et consommant moins de tokens que les solutions rivales
- Outil open-source gratuit qui échappe aux modèles SaaS coûteux et aux tarifs volatiles (nombreux utilisateurs frustrés par le passage de tarifications forfaitaires à paiement à l'usage)
- Architecture bien pensée avec gestion parallèle de grandes modifications, modules de localisation et réflexion pour améliorer la précision et la qualité des retours
- Intégration facile dans les workflows de développement via CLI et agents (Claude Code, Codex), permettant d'automatiser les revues sans infrastructure complexe

**Avis négatifs** :
- Taux de faux positifs très élevé (~87%) détecté sur benchmark, ce qui risque d'entraîner que les développeurs ignorent les alertes, minant la crédibilité de l'outil
- Fonction de rappel insuffisante (~74%) : ne détecte pas tous les problèmes importants, mettant en question son efficacité réelle face aux outils payants comme CodeRabbit
- Fichiers de règles en chinois rendant l'outil difficile d'accès sans traduction externe, et incompatibilité avec les modèles GPT-5.x (erreur hardcodée sur les paramètres)
- Ne remplace pas la revue humaine : l'absence d'humains dans la boucle crée du « review theater » ; seule la revue par d'autres développeurs apporte la socialisation des connaissances et crée une trace d'audit
- Absence d'évidence empirique que l'utilisation de modèles différents pour coder et reviewer améliore réellement les résultats, contrairement aux hypothèses anthropomorphes courantes
