## Discussion sur Hacker News (96 commentaires)

**Avis positifs** :
- L'abstraction unifiée d'Apple permet aux développeurs d'écrire une seule fois contre une API standardisée et de basculer facilement entre modèles locaux, Claude, ou d'autres fournisseurs selon les besoins sans refactoriser le code
- Apple offre une approche intelligente en commoditisant les LLM tout en maintenant le contrôle du matériel optimal pour l'IA, positionnant les modèles frontière comme des services interchangeables plutôt que comme des produits verrouillés
- Cette stratégie bénéficie à la confidentialité des utilisateurs : les requêtes vont directement d'Anthropic à l'app (pas de transit par Apple), et Apple ne prend pas de commission directe sur l'utilisation de Claude
- Le framework simplifie la gestion des aspects techniques complexes (streaming, tool calling, structured output) et maintient la continuité de session même lors de commutation dynamique entre fournisseurs

**Avis négatifs** :
- Apple configure une couche d'abstraction pour mieux monétiser via l'App Store (30% de commission sur les abonnements/achats intégrés) tout en gardant une apparence de neutralité, ce qui rappelle la stratégie de Google avec les moteurs de recherche
- Le modèle expose des problèmes d'UX problématiques : demander aux utilisateurs d'entrer leur propre clé API est une barrière trop haute, et le proxying côté serveur révèle toutes les données utilisateur au proxy
- L'absence de déduplication des modèles entre applications pose un vrai problème : plusieurs apps téléchargeant les mêmes modèles locaux bloatera les appareils, et Apple n'offre pas de solution de cache partagé ou de gestion centralisée des modèles
- Apple contrôle finalement l'écosystème via cette API : elle facilite la transition vers ses propres modèles propriétaires une fois qu'ils seront suffisamment performants, verrouillant davantage les développeurs dans son environnement
- Les exemples de code montrés ressemblent davantage aux APIs Anthropic originales qu'à une véritable abstraction Apple, créant de la confusion sur ce que le framework apporte réellement
