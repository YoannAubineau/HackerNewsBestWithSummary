## Résumé de l'article

Le document établit les règles d'engagement pour les assistants IA (ChatGPT, Claude, GitHub Copilot, etc.) accompagnant les étudiants du cours CS336, un cours intensif de programmation en Python/PyTorch. Les assistants doivent servir d'outils pédagogiques qui guident l'apprentissage sans résoudre directement les assignments.

- **Rôle autorisé** : expliquer des concepts, pointer vers les ressources du cours, examiner le code étudiant, poser des questions de débogage, expliquer des messages d'erreur, suggérer des sanity checks et des tests.
- **Actions interdites** : écrire du code de solution, compléter les sections TODO, refactoriser le code vers une solution finie, implémenter les composants principaux (tokenizers, transformers, optimizers, etc.), pointer vers des implémentations tierces.
- **Approche recommandée** : poser des questions clarifiantes, référencer les matériaux du cours plutôt que donner des réponses directes, suggérer les prochaines étapes sans les implémenter, dialoguer autour du code pour identifier les zones d'amélioration.
- **Principes clés** : l'objectif est que les étudiants apprennent en faisant, pas en regardant une IA générer des solutions ; en cas de doute, orienter vers l'équipe pédagogique ou les heures de bureau.

## Discussion sur Hacker News (96 commentaires)

**Avis positifs** :
- Reconnaître que l'IA est inévitable et adapter les évaluations plutôt que de les interdire : augmenter la difficulté, exiger des projets substantiels, des examens oraux et des tests en conditions contrôlées pour vérifier la compréhension réelle.
- Une approche équilibrée qui prépare les étudiants à la réalité professionnelle où l'IA est omniprésente, tout en les guidant vers un usage responsable comme outil d'apprentissage plutôt que contournement.
- Les directives aident à établir des normes claires et à communiquer l'intention pédagogique, même si l'application n'est pas techniquement forcée ; les étudiants qui cherchent vraiment à apprendre bénéficient d'un cadre explicite.
- Des alternatives éprouvées existent (examens écrits sans ressources, examens oraux, projets pratiques en laboratoire) et le modèle Stanford d'honneur académique avec pénalités sévères peut fonctionner pour cette population spécifique.
- Contrairement aux interdictions générales, cette approche est un pas dans la bonne direction et ne bride pas l'innovation, reconnaissant que les étudiants utiliseront de toute façon l'IA.

**Avis négatifs** :
- Ces directives sont fondamentalement inapplicables : les étudiants peuvent facilement modifier les fichiers CLAUDE.md ou utiliser d'autres modèles externes, rendant l'exercice symbolique et inefficace.
- Straddling the middle ground sans prises véritables est contre-productif : soit permettre l'IA avec des évaluations plus difficiles, soit l'interdire strictement avec examens traditionnels, mais pas cette approche mitigée qui satisfait personne.
- Le risque que les étudiants utilisent l'IA passivement comme tuteur superficiel sans véritable apprentissage approfondi reste majeur ; les directives seules ne changent pas le comportement incité par les pressions académiques.
- Stanford, avec ses 50 milliards de dollars d'endowment, pourrait faire bien mieux que ces directives vagues ; une véritable innovation éducative nécessiterait des changements structurels et une répartition claire des modules selon leurs objectifs.
- Le système reste vulnérable au déshonneur académique généralisé : même avec un code d'honneur, les étudiants peuvent tricher sans détection, particulièrement pour ceux sans intention réelle d'apprendre.
