## Résumé de l'article

Le nouveau Outlook de Microsoft, construit sur WebView2 (le moteur de rendu Chromium d'Edge), souffre d'un délai significatif à l'ouverture des emails via les notifications Windows 11 : environ dix secondes contre une réponse instantanée avec Outlook Classic, l'ancienne application Win32 native. Cette lenteur est structurelle et difficile à corriger, due à l'architecture web-based du nouvel Outlook.

- Le nouveau Outlook fonctionne en dix processus distincts (WebView2 Manager, GPU, Service Worker, etc.) consommant 490-636 MB de RAM au repos, contre un seul processus pour Outlook Classic utilisant 117-148 MB
- Cliquer sur une notification Windows 11 oblige le nouvel Outlook à initialiser sa couche web, s'authentifier, charger le fil de messages et le rendre, tandis qu'Outlook Classic affiche directement l'email concerné
- Microsoft a reporté la migration forcée des entreprises vers le nouvel Outlook de avril 2026 à mars 2027, reconnaissant que l'app n'est pas prête pour tous les workflows
- Les améliorations en 2024-2026 (recherche améliorée, calendriers partagés, unified inbox à venir) ne résolvent pas les limites fondamentales de l'architecture WebView2
- Outlook Classic reste disponible jusqu'en avril 2029 ; seule une migration vers WinUI (code natif Windows) pourrait résoudre les problèmes de performance actuels

## Discussion sur Hacker News (240 commentaires)

**Avis positifs** :
- La version web d'Outlook (outlook.office.com) fonctionne nettement plus vite que le client lourd et s'exécute bien sur Linux, rendant le client de bureau souvent superflu pour la plupart des besoins professionnels
- D'autres clients web comme Fastmail offrent des performances acceptables, prouvant qu'il est possible de développer des applications web email performantes sans compromettre la vitesse
- WebView2 n'est pas intrinsèquement responsable des problèmes de performance ; d'autres applications natives de Microsoft ont aussi ralenti, suggérant un problème plus profond de conception organisationnelle plutôt que technologique

**Avis négatifs** :
- Le nouveau Outlook consomme 4 à 5 fois plus de RAM au repos (490-636 MB vs 117-148 MB) sans bénéfice fonctionnel, tandis que des tâches simples comme ouvrir une fenêtre de réponse prennent 10 secondes
- Microsoft a systématiquement sacrifié la performance pour ajouter des fonctionnalités inutiles : les notifications de réunion affichent un écran de chargement de 5 secondes, la calculatrice de Windows prend plusieurs secondes à démarrer, et même le menu Démarrer est devenu lent
- Le nouveau Outlook manque de parité fonctionnelle avec la version classique, intègre des annonces publicitaires même pour les utilisateurs payants, et force l'envoi des identifiants IMAP en clair aux serveurs Microsoft au lieu de gérer directement IMAP
- La culture d'ingénierie chez Microsoft privilégie la compatibilité rétroactive et le court terme plutôt que l'excellence technique ; les équipes sont poussées à ajouter des fonctionnalités inutiles pour justifier leur existence au lieu de résoudre les vrais problèmes
- Le problème est systémique chez Microsoft : Windows 11, Teams, l'Explorateur de fichiers et d'autres applications natives se sont tous dégradés, indiquant une détérioration organisationnelle plus large qu'une simple question technologique
