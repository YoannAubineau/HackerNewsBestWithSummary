## Résumé de l'article

SQL to ER Diagram est un outil open source gratuit qui convertit des schémas SQL en diagrammes entité-relation interactifs directement dans le navigateur, sans envoyer de données vers un serveur.

- Parse les instructions CREATE TABLE et ALTER TABLE pour PostgreSQL, MySQL, SQLite et SQL Server, en supportant clés primaires, clés étrangères et contraintes
- Permet de glisser-déposer les instructions SQL, d'arranger les tables interactivement, d'ajouter des notes et de renommer les éléments par double-clic
- Exporte les diagrammes en PNG haute résolution ou SVG, avec option de sauvegarde du projet complet ou génération d'un lien partageable
- Fonctionne entièrement dans le navigateur sans installation, compte ni inscription requise
- Garantit la confidentialité en exécutant tout localement : le schéma SQL ne quitte jamais l'ordinateur de l'utilisateur

## Discussion sur Hacker News (54 commentaires)

**Avis positifs** :
- L'outil offre une excellente expérience utilisateur sans barrières d'entrée (pas de signup, pas de paywall, données sécurisées localement) répondant aux frustrations des outils existants.
- L'implémentation technique est impressionnante : utilisation de canvas pour la performance (centaines de tables sans ralentir), parseur SQL preservant la mise en forme, partage par URL sans backend, et le tout en ~32KB sans framework.
- La convivialité mobile est exceptionnelle avec pan/zoom/édition fluides, même sur Safari, représentant un niveau de raffinement rarement vu.
- Le code source est élégant et représente une solution simple et distillée pour un problème complexe, méritant des éloges pour la qualité de conception.
- Les utilisateurs apprécient le projet comme solution moderne et bien exécutée aux anciens outils comme MySQL Workbench, correspondant à un besoin réel.

**Avis négatifs** :
- La terminologie 'ER diagram' est techniquement contestable car les entités conceptuelles diffèrent fondamentalement des tables SQL, l'outil générant plutôt des schémas visuels de tables.
- Certaines limitations fonctionnelles existent : pas de sélection multiple de tables, lignes courbes non personnalisables, manque de filtrage des connexions (problématique pour les applications multi-locataires avec nombreuses références partagées).
- L'optimisation du layout contient des chevauchements visibles de lignes de relation, suggérant que l'arrangement automatique n'est pas optimal même pour les exemples fournis.
- Il existe déjà des solutions concurrentes établies (Azimutt, dbdiagram, DBeaver) offrant des fonctionnalités comparables, et l'outil manque de fichier LICENSE dans le dépôt.
- Les limitations pratiques d'URL (bien que suffisantes en théorie) pourraient être contraignantes pour les très grands schémas, malgré les workarounds existants.
