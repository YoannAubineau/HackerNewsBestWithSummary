## Résumé de l'article

JSON-LD (JSON Linked Data) est un format pour ajouter des données structurées aux pages web, permettant aux moteurs de recherche et crawlers de mieux comprendre le contenu et d'améliorer l'affichage des résultats. Cet article détaille comment implémenter JSON-LD sur un site personnel en utilisant le schéma Schema.org et décrit les nœuds essentiels (WebSite, Person, WebPage, BlogPosting, etc.) avec des exemples à adapter.

- JSON-LD s'ajoute dans la balise `<head>` via une balise `<script type="application/ld+json">` ; les crawlers spécialisés comme Googlebot le lisent sans que le navigateur l'exécute
- Chaque nœud JSON-LD possède un `@type` (sa nature), un `@id` (identifiant unique, généralement une URL avec hash), et des propriétés décrivant ses attributs
- Les nœuds essentiels pour un site personnel sont WebSite (métadonnées globales), Person (profil de l'auteur), ProfilePage (page à propos), WebPage (pages individuelles) et BlogPosting (articles)
- BreadcrumbList améliore l'affichage du chemin dans les résultats de recherche ; Blog et CollectionPage structurent les contenus de type liste ou index
- Même les sites statiques bénéficient d'une implémentation minimale (WebSite, ProfilePage, Person sur la page racine) ; les propriétés comme `sameAs` aident à la désambiguïsation et aux graphes de connaissances

## Discussion sur Hacker News (60 commentaires)

**Avis positifs** :
- JSON-LD est pratique et facile à mettre en œuvre comparé à RDFa, même si cela implique une redondance avec le HTML, car la cohérence des métadonnées à travers multiples formats est complexe
- JSON-LD remplit un besoin distinct du HTML sémantique en exprimant des métadonnées structurées (date de création, auteur, tags) que les moteurs de recherche exploitent effectivement
- La flexibilité de JSON-LD permet de combiner plusieurs vocabulaires (schema.org, FOAF, Dublin Core) dans un même contexte pour exprimer des propriétés non couvertes par un seul standard
- Pour les sites e-commerce et professionnels, JSON-LD aide les moteurs de recherche (cartes, horaires, menus) et les constructeurs de sites l'utilisent pragmatiquement malgré les limites théoriques

**Avis négatifs** :
- L'ajout de JSON-LD crée une redondance inutile : les données doivent être exprimées deux fois (HTML et JSON-LD script tag) alors que RDFa ou les microformats éviteraient cette duplication
- La prolifération de standards de métadonnées (JSON-LD, RDFa, Microdata, OpenGraph, Dublin Core) crée de la confusion; OpenGraph est d'ailleurs plus largement supporté pour les aperçus riches que JSON-LD
- L'utilité réelle de JSON-LD pour le référencement s'érode avec les LLM : Google génère désormais des résumés LLM sans diriger le trafic vers les sources, rendant ces efforts obsolètes et impactant peu la visibilité réelle
- La Semantic Web a échoué après 25 ans : les métadonnées deviennent rapidement obsolètes, sont souvent inexactes, peu de sites les implémentent, et les acteurs malveillants les falsifient; les LLM génératives seraient plus efficaces pour extraire les données
- JSON-LD représente une exploitation asymétrique : les créateurs de contenu fournissent du travail gratuit pour enrichir les index des géants du web sans bénéfice visible, particulièrement avec l'essor des LLM-scraping
