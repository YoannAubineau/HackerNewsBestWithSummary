## Résumé de l'article

Bash dispose d'une fonctionnalité interne permettant d'ouvrir des sockets TCP par la redirection /dev/tcp, utile pour tester la connectivité dans des conteneurs minimalistes dépourvus d'outils réseau. Cela facilite les vérifications rapides sans installer de dépendances supplémentaires.

- La syntaxe `exec 3<>/dev/tcp/host/port` ouvre une socket TCP et assigne un descripteur de fichier (3) pour lire/écrire des données
- Les requêtes HTTP s'écrivent manuellement avec `printf` en respectant le format HTTP/1.1, notamment l'en-tête `Connection: close` pour forcer la fermeture de la connexion
- /dev/tcp n'est pas un vrai fichier disque mais une redirection interne Bash interprétée par le shell lui-même
- Cette technique ne gère pas les redirects, la compression, TLS, les réponses chunked ou autres fonctionnalités que curl automatise ; elle reste limitée au HTTP en clair
- La fonctionnalité est spécifique à Bash (compilée avec `--enable-net-redirections`) ; elle ne fonctionne pas avec sh/POSIX ou zsh, et certains systèmes Debian historiquement l'ont désactivée

## Discussion sur Hacker News (160 commentaires)

**Avis positifs** :
- Pratique pour les diagnostics rapides de connectivité réseau dans les conteneurs Docker minimalistes (Python, Alpine) dépourvus de curl/wget/netcat.
- Utile pour les vérifications de santé (healthchecks) et tests locaux sans dépendre de binaires externes ou de la réinstallation d'images.
- Démontre l'accessibilité des protocoles texte (HTTP, SMTP) et révèle comment fonctionnent réellement les systèmes réseau.
- Alternative valable lorsque les outils standard sont explicitement bloqués par les restrictions de sécurité ou l'absence de binaires disponibles.

**Avis négatifs** :
- Bash n'implémente pas vraiment HTTP : c'est seulement un socket TCP brut. Un vrai client HTTP doit gérer les proxies, encodages, redirections, HTTPS et autres complexités que curl maîtrise.
- Cette fonctionnalité n'est pas POSIX et désactivée par défaut dans Debian (avant 2009). Les scripts ne sont pas portables sur sh, zsh ou dash.
- Pour les cas sérieux en production, utiliser curl/wget (moins de 10 MB sur Alpine) reste préférable. Les images minimales (FROM scratch) sacrifient le debugging et augmentent les frictions opérationnelles.
- Le problème de la réduction de surface d'attaque via les images distroless est surévalué : les CVE dans les os components peu utilisés et la possibilité d'attacher des debug containers rendent cette approche contestable.
