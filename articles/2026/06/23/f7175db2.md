## Résumé de l'article

TikZ Editor est un éditeur WYSIWYG (« ce que vous voyez est ce que vous obtenez ») gratuit et open source qui permet de créer et modifier des diagrammes TikZ pour LaTeX en les dessinant directement plutôt qu'en écrivant du code. L'application met à jour le code TikZ en temps réel à mesure que vous déplacez les éléments, tout en préservant la formatation existante.

- Manipulation visuelle des éléments : déplacer, redimensionner et faire pivoter les nœuds et chemins directement sur le canevas, avec mise à jour instantanée du code sans recompilation
- Outils de dessin complets : création de nœuds, formes, flèches, courbes, grilles, rectangles, ellipses, cercles, et support des boucles \foreach
- Édition multi-figures : ouverture de fichiers .tex complets avec plusieurs figures TikZ, navigation par aperçus et compréhension des macros personnalisées
- Fonctionnalités d'alignement et distribution : sélection multiple d'objets, groupement en scopes, snapping à la grille, guides personnalisables et outils d'alignement
- Assistance IA sur desktop : intégration optionnelle de Codex (ChatGPT) pour demander des modifications directes des figures ; export en SVG, PNG, PDF ou LaTeX autonome

## Discussion sur Hacker News (52 commentaires)

**Avis positifs** :
- Comble un besoin longtemps attendu : les chercheurs et étudiants en STEM demandaient depuis des années un éditeur WYSIWYG pour TikZ, particulièrement comparé à des alternatives anciennes comme Xfig
- Préserve et améliore le code source existant : contrairement à Inkscape ou d'autres outils, l'éditeur maintient la lisibilité du code TikZ original lors de modifications, évitant le code généré automatiquement
- Usage judicieux des agents IA : le projet démontre une application pratique des LLM pour automatiser des tâches répétitives (génération de code TikZ) que les humains ne voudraient pas faire manuellement
- Licence permissive et extensibilité : en MIT et open source, l'outil peut être intégré dans d'autres applications (Obsidian, VS Code, etc.) et utilisé comme bloc de construction par d'autres projets
- Interface intuitive avec possibilités avancées : le support des nœuds, des lignes avec points d'attache, et la capacité à éditer du code TikZ complexe existant rend l'outil accessible aux débutants tout en restant puissant

**Avis négatifs** :
- Génération de code suboptimale : le code produit utilise systématiquement des coordonnées absolues au lieu d'exploiter les fonctionnalités natives de TikZ (ancres, positionnement relatif, alignement), rendant le code moins élégant et maintenable
- Support limité de TikZ avancé : pgfplots, tikzcd, CircuiTikZ et autres extensions spécialisées ne sont pas supportées, nécessitant une infrastructure d'add-ons compliquée à cause des conflits de licence
- Problèmes de compatibility sur certaines plateformes : des utilisateurs Linux signalent des dysfonctionnements graphiques (grille non-carrée), indiquant que le support multi-plateforme n'est pas complètement robuste
- Pas d'équivalent Typst/Cetz : les utilisateurs de Typst (en croissance) sont exclus, et l'auteur n'envisage pas de support dans l'immédiat faute de familiarité avec Cetz
- Limitations UI pour les fonctionnalités complexes : des concepts TikZ comme le positionnement relatif ou le changement contextuel d'ancres n'ont pas d'interface intuitive définie, obligeant l'édition manuelle du code source
