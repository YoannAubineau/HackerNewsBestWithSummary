## Résumé de l'article

L'auteur a mis en place OpenCode, un environnement de développement web accessible et multi-appareils, pour gérer son homelab composé d'une douzaine de stacks Docker. OpenCode est configuré pour pousser les modifications vers Git en tant que branches de fonctionnalités, l'auteur validant les PR avant déploiement automatisé via GitOps.

- OpenCode fonctionne comme serveur avec interface web, terminal intégré et support des sessions persistantes synchronisées entre appareils
- L'IA est maintenue derrière une revue PR : OpenCode propose les changements, l'auteur approuve et fusionne lui-même, préservant un contrôle sur le code déployé
- OpenCode a accès au serveur Git et bénéficie de droits root limités à la VM d'exécution, sans accès direct aux services en production (blast radius réduit)
- L'auteur utilise cette plateforme pour automatiser les mises à jour de conteneurs (lecture de release notes, ajout de healthchecks) et gérer l'infrastructure réseau sur multiple appareils
- Limitation actuelle : Forgejo Actions n'expose pas les logs de tâches via API publique, entravant la boucle de feedback rapide pour diagnostiquer les erreurs de tests ou linters

## Discussion sur Hacker News (47 commentaires)

**Avis positifs** :
- Plusieurs développeurs confirment qu'ils expérimentent des configurations similaires avec des modèles locaux (Qwen, Gemma) et des workflows d'automatisation (n8n, Argo, K3s), validant l'approche décrite
- Les modèles open-source comme Qwen 27B/35B-A3B peuvent gérer les tâches agentic avec une quantification 4-bit sur des systèmes domestiques, contrairement à l'idée qu'il faudrait des ressources massives
- L'intégration de Claude avec les systèmes personnels simplifie considérablement la maintenance et étend les capacités opérationnelles des homelabs
- L'approche GitOps + IA pour déployer automatiquement les changements est pratique et plusieurs outils existent pour l'implémenter (Arcane, Doco CD, Materia, Komodo)

**Avis négatifs** :
- Les modèles locaux ne sont pas encore tout à fait prêts pour les workflows agentic sans investissement substantiel, malgré les améliorations récentes
- Les templates de chat des modèles open-source sont très variables et peu fiables selon l'utilisation spécifique, causant des problèmes majeurs
- Beaucoup d'utilisateurs rapportent que les bénéfices réels sont marginaux pour les tâches complexes—le débogage manuel est souvent plus rapide et moins sujet aux erreurs
- L'article manque de clarté sur le problème réel résolu: c'est surtout un assemblage technique sans justification pratique convaincante
- Les ressources requises (VM, RAM) pour exécuter OpenCode localement et construire des projets rapidement restent prohibitives pour beaucoup d'utilisateurs
